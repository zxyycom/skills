# 2026-07-18 - 让长期决策形成和冲突成为核心触发

## 索引摘要
- 目的: 在真正形成长期判断或发生冲突时恢复决策上下文，避免普通执行误触发。
- 背景: 以“任务可能改变长期行为”作为触发条件会把普通执行也纳入范围，而逐项枚举维护操作会削弱真正需要主动恢复上下文的决策时刻。
- 决策: `decision-records` 用于长期决策管理；当用户正在形成可能产生长期影响的决定，或拟议决定与既有决定冲突时调用，触发后恢复相关当前决策并处理偏离，入口不枚举记录、查询或归档等自然操作。

## 目的
- 在真正形成长期判断或发生冲突时恢复决策上下文，避免普通执行误触发。

## 背景
- 等待用户主动要求记录或回放，会错过用户专注推进工作、尚未意识到需要恢复既有方向的决策时刻。
- 仅凭任务可能影响长期行为就触发，会把尚未出现长期取舍的普通执行也纳入范围，增加无效介入。
- Skill frontmatter 的 `description` 是主要路由入口，正文只有触发后才会加载，因此入口应明确能力领域和真正需要主动介入的条件。
- 记录、查询、回放和归档本身已经属于长期决策管理，不需要在入口逐项枚举。

## 决策
- 采用: `decision-records` 的入口先用“用于长期决策管理”声明能力领域。
- 采用: 用户正在形成可能持续影响后续行为、owner、边界、兼容性、风险处理或验收方式的决定时调用。
- 采用: 拟议决定与既有决定冲突时调用；触发后先恢复相关当前决策，再区分对齐、一次性例外、长期修订和一致性问题。
- 采用: 记录、查询、回放、修订和归档等直接决策操作由能力领域自然覆盖，不在 `description` 中重复列举。
- 采用: 主动恢复不等于自动写入；只有明确确认的长期取舍才进入当前索引。
