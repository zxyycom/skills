## 目的
- 让 `skills/investigation-report/` 形成的每份报告保存结果产生时实际使用的背景、调查动因和调查目标，使未来读者能够重放判断输入，不必补入默认前提或拼接其他报告。
- 保留同一核心问题在不同阶段、对象状态、环境、假设或调查目标下形成的完整认识；结果相同时也能作为跨背景比较证据。
- 让最新报告承担当前汇报口径，较早报告保存当时认识和适用边界；同时保持稳定主题路径、轻量索引、无顶部结果摘要和明确沉淀才触发的既有边界。

## 背景
- 前序决策已经用追加完整独立报告取代顶部概述加增量调查段，解决了新结果依赖旧报告或原对话才能解释的问题；其固定核心是 `背景`、`起因` 和 `调查结果`。
- 后续审阅进一步明确，报告还要保存每轮认识形成时的判断输入。省略当时已确认的事实、采用的假设、尚未解决的未知、关键约束或对象基线，会让后续 agent 把未写出的默认前提带入新环境，造成认识漂移或遗漏。
- 主题级核心问题只提供稳定的问题链身份；调查动因只解释为什么此时开展调查。二者都不能替代每轮具体希望查清什么、范围到哪里以及结果准备支持什么判断。
- 相同结果出现在不同背景下并非无效重复。背景差异可能说明结果具有跨环境稳定性，也可能暴露尚未解释的共同条件；分别保存完整输入后才能作出这种比较。
- 后来被推翻的假设仍可能准确表达当时认识，应作为历史快照保留；没有准确记录当时认识的错写、漏写或定位错误则不构成新认识。为避免重新退化为填表，只把此前容易被隐含的报告级调查目标新增为固定语义。

## 决策
- 采用: 一个稳定主题文件持续承接同一核心问题；能够独立调查、交接和结束的新问题创建兄弟文件。
- 采用: 每份完整报告是一份形成于特定时点的认识快照，固定依次包含非空的 `背景`、`调查动因`、`调查目标` 和 `调查结果`，并能脱离原对话和同一主题中的其他报告独立阅读、汇报和复核。
- 采用: `背景` 保存解释本轮结果所必需的当时项目处境、阶段、对象状态和现有机制，并按实际影响写清已确认事实、当时假设或工作模型、当时未知、关键约束以及版本、环境、样本、时间范围或工作区状态等条件性基线。事实、假设和未知必须明确区分，后来的认识不得倒写成当时背景。
- 采用: `调查动因` 写触发本轮调查的观察、变化、疑问、复查需要或外部要求，不把尚未证明的根因写成动因；`调查目标` 写本轮具体要查清的问题、适用范围和必要边界，不能只由主题级核心问题或动因隐含。
- 采用: `调查结果` 先写本轮实际发现、适用边界或明确未知，并能够回指本轮背景、目标和证据。范围与方法、证据、推断、结论、冲突、未知和下一步继续作为按需支撑内容，不替代四个固定核心章节。
- 采用: 当影响结果解释的背景、调查动因、调查目标、证据范围或解释发生实质变化，并形成能够独立汇报的认识时，在原主题文件末尾追加新的完整报告。即使调查结果与前一份一致，只要形成结果的背景或目标不同，也保留为独立报告。
- 采用: 后来取得的新事实或新理解改变了早先真实采用的背景、假设、结果、推断或适用边界时，通过追加报告表达演进并保留旧报告。旧报告没有准确记录当时实际认识的错别字、格式、链接、定位、误写或漏写可以原地更正；仅改变是否继续调查时可以只更新状态。
- 采用: 最新一份完整报告是当前认识的报告正文 owner，较早报告承接历史回放和跨背景比较。主题文件顶部只保留核心问题、状态和最新报告时间，不另存可能与正文漂移的当前结果摘要。
- 采用: Git commit hash 只在结果依赖已提交快照时作为条件性基线；未提交改动影响结果时同时记录相关 diff、内容或工作区状态，与 Git 快照无关时不要求 hash。
- 采用: 检查器机械验证调查信息、完整报告外壳、四个固定核心章节、形成时间顺序、最新报告时间和索引投影；背景是否完整、事实与假设是否正确区分、目标是否明确以及不同报告是否具有独立比较价值继续由人工审阅。
- 采用: `investigation-report` 继续只在用户明确要求记录、沉淀、创建、更新或审阅调查报告时触发，不扩张为普通调查的主动记录建议。
