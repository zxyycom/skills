## 目的
- 让 `skills/investigation-report/` 指导 agent 在调查进行期间持续形成一份能够直接汇报当前结果、帮助新读者恢复必要上下文、并让后续调查者从已有证据和未知继续推进的报告。
- 让报告保留每轮认识形成时的背景、方法、证据、结果、推断和时间，同时由顶部概述提供当前汇报口径。
- 保持一种持久化报告格式和轻量机械检查，不把能力扩张为通用调查路由、主动记录建议或复杂生命周期管理。

## 背景
- 调查报告的最终消费不只是归档。维护者需要快速知道当前认识，第一次接触问题的人需要降低恢复背景和证据链的成本，后续调查者还需要知道每轮做过什么、当时怎样理解以及下一步缺什么。
- 只把写作放在调查完成之后，容易让调查方法与最终报告脱节；把完整调查流程作为独立目标，又会使 skill 偏离稳定产出报告。更合适的行为是让每轮调查以形成可接续报告段为完成证据。
- “结果可能长期有用”覆盖面过宽，会让普通排障、问答和临时调查频繁收到记录建议。库调研、竞品调研、机制与性能分析、实际执行逻辑和并发研究可以作为少量常见报告对象，但仍应由用户明确要求沉淀。
- 报告级开始时间不能解释滚动调查中不同段落的背景。需要同时区分报告第一次形成可用内容的时间、每轮认识形成时间和报告最近实质更新时间，并让实验、日志和外部来源继续保留各自观测时间。

## 决策
- 采用: `investigation-report` 保持独立分发，以产出与流程型能力为主；它组织调查是为了稳定形成报告，不承接普通调查的通用执行方法。
- 采用: 只有用户明确要求记录、沉淀、创建、更新或审阅调查报告时使用。能力所需的库调研、竞品调研、机制与性能分析、实际执行逻辑和并发研究作为常见适用内容，不使 skill 在普通调查中主动建议或创建报告。
- 采用: 主流程按报告框架、本轮调查、证据判断、调查段形成、当前概述刷新、质量审阅和结构校验展开。每轮调查以形成一段能够恢复背景、方法、证据、结果、推断、未知和接续入口的内容为完成证据。
- 采用: 报告继续使用语义路径和固定 `调查概述`，并固定第二个二级标题为 `调查记录`。每轮调查使用自由命名的三级标题，首个内容记录秒级带时区的 `形成时间`，段内表达方式由问题决定。
- 采用: 概述中的 `开始时间` 改为 `首次形成时间`，表示报告第一次形成可汇报、可接续调查段的时刻；`最近更新时间` 投影最新实质变化。每个调查段另有形成时间，证据自身仍保留观测或访问时间。
- 采用: 索引继续只投影标题、核心问题、状态和最近更新时间；检查器继续默认全量并支持主题与报告筛选，只机械校验固定概述、调查记录外壳、形成时间和索引投影，不判断报告语义质量。
