# 2026-06-27 - 细化 AGENTS 中的决策记录入口

问题:
- AGENTS 的决策记录段落只有一句话, 能指出写入位置, 但不足以提醒维护者记录决策的判断边界。
- 过细的模板和字段说明又会让 AGENTS 重新变成维护细则, 与“只保留稳定项目约定”的决定冲突。
- 用户要求 AGENTS 中也细化决策记录。

决策过程:
- `docs/decisions/` 已经承接完整目的、范围、写法和历史记录。
- AGENTS 只需要提供稳定入口: 什么情况要记录、记录要回答什么、不记录什么、旧决定如何处理。
- 模板和详细示例继续由 `docs/decisions/maintenance.md` 承接, 避免重复。

决定:
- 将 AGENTS 的决策记录段落扩展为四条稳定规则。
- AGENTS 说明需要记录的范围、记录应包含的问题/取舍/决定/影响、不需要记录的低风险事项, 以及替代旧判断时追加新记录。
- 不在 AGENTS 中复制决策记录模板。

影响:
- AGENTS 对决策记录的指导更清楚, 但仍保持项目级稳定约定。
- 后续需要调整记录模板或示例时, 只修改 `docs/decisions/maintenance.md`。

验证:
- AGENTS 的决策记录小节已扩展为四条规则。
- AGENTS 仍未包含完整模板或历史细节。
