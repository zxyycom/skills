# 2026-06-27 - AGENTS 只保留稳定项目约定

问题:
- AGENTS 如果混入文件级职责、详细检查项和频繁变化的 skill 行为细节, 后续每次调整 skill 都可能需要同步修改 AGENTS。
- 这会让 AGENTS 从项目级约定变成二级规则索引, 增加重复和维护成本。
- 用户明确指出 AGENTS 不应过分掺杂细节。

决策过程:
- README 已经承担项目基础介绍和目录说明。
- skill 本体和 references 承担具体行为、读取策略、规则和完成检查。
- 决策记录承担长期取舍和修改历史。
- AGENTS 更适合作为稳定的维护边界说明, 只保留不容易随具体规则变化的约定。

决定:
- 精简 AGENTS, 删除文件级 owner 清单、详细决策触发清单和验证清单。
- AGENTS 只保留项目边界、稳定维护原则和决策记录入口。
- 以后只有项目级维护边界变化时才修改 AGENTS；具体规则变化优先修改 README、skill 本体、references 或决策记录。

影响:
- AGENTS 后续改动频率应降低。
- 具体文件职责和 skill 行为不再在 AGENTS 中重复展开。
- 需要追溯为什么这样收敛时, 查看本条决策。

验证:
- AGENTS 内容已收敛为项目边界、维护原则和决策记录三段。
- AGENTS 不再包含文件级 owner 清单和详细验证清单。
