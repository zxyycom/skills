# 2026-06-27 - 取消固定读取清单

问题:
- AGENTS 中的固定读取清单把小型文档项目写成了重流程。
- 本项目文件数量少, 直接扫一遍当前项目文件和相关上下文成本很低, 不需要按场景维护多条读取规则。
- 固定清单容易让后续维护者关注流程执行, 而不是当前文档实际需要解决的问题。

决策过程:
- README 已经说明项目结构和各文件用途。
- AGENTS 的 owner 是维护原则, 不需要承担细粒度操作清单。
- 对这个项目来说, 保留“快速扫当前项目文件和相关上下文”的原则, 比维护固定读取顺序更符合实际使用方式。

决定:
- 删除 AGENTS 中的固定读取清单。
- 在修改原则中保留轻量约定: 项目规模小, 开始修改前快速扫当前项目文件和相关上下文即可。
- 仓库内 skill 包仍是当前项目来源, 仓库外副本不作为默认修改目标。

影响:
- 后续维护不需要按文档类型套用读取清单。
- 修改前仍应理解当前项目上下文, 但由任务范围自然决定查看哪些文件。
- AGENTS 更聚焦于项目身份、内容 owner、修改原则、决策记录和验证。

验证:
- AGENTS 不再包含固定读取清单章节。
- AGENTS 的修改原则包含轻量上下文确认规则。
- 决策记录追加本条取舍。
