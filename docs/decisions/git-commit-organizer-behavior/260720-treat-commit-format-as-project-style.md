## 目的
- 让 `git-commit-organizer` 在跨项目复用时保持稳定、有明确偏好的提交表达，并让使用者清楚理解安装或调用该 skill 所采用的默认行为。

## 背景
- 该 skill 面向已经大致了解其作用、在理解影响后主动安装或调用的使用者，不需要承担面向任意仓库和任意熟练度的通用 Git 教学。
- 固定提交类型、中文全角冒号、中文标题、正文 bullet 和语义拆分倾向共同构成它的核心价值。
- 自动推断并迁就每个仓库的既有提交格式会削弱这套风格的稳定性；但无关 staged 内容等问题会直接影响实际提交范围，仍需要明确门禁。

## 决策
- 采用: 将 `git-commit-organizer` 定位为可跨项目复用、带明确偏好的项目级提交风格；安装或显式调用表示使用者希望采用这套默认风格。
- 采用: 行为入口继续明确规定类型、中文全角冒号、中文标题和正文 bullet，用户或项目对当前任务给出更具体要求时按该要求调整。
- 采用: 优先维护日常提交整理中的语义、范围和结果复核，只补足会影响提交正确性的必要门禁。
- 不采用: 不把自动适配任意仓库风格或覆盖低频 Git 场景作为主流程目标。
