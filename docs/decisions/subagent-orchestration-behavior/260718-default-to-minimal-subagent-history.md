# 默认以最小历史派发子代理

## 索引摘要
- 目的: 让子代理只获得完成任务所需的上下文，降低无关历史干扰并保持工具无关。
- 背景: 入口要求最小上下文，但缺少可映射到不同工具的历史选择规则；依赖默认值会引入噪音，绑定字段又降低复用性。
- 决策: 派发前按当前工具能力确定最小充分的历史范围; 可控时显式配置, 默认不继承主线程对话, 必要时只继承最小有限范围, 仅在完整对话本身是不可替代输入时主动选择全量继承, 具体字段和值由工具接口承接。

## 目的
- 让子代理只获得完成任务所需的上下文，降低无关历史干扰并保持工具无关。

## 背景
- `skills/subagent-orchestration/SKILL.md` 已经要求只传递完成子任务所需的最小上下文, 但没有把该原则落实为工具无关的历史选择规则。
- 不同子代理工具使用不同字段、合法值和默认行为表达历史继承。绑定单一工具字段会使 skill 难以复用, 依赖默认行为又可能复制与子任务无关的主线程历史。
- 过量继承会增加重复理解成本, 同时减少子代理留给调查、实现和验证的上下文空间。
- 子代理可以从其可访问的稳定材料恢复项目规则和任务事实; 主代理也可以在任务消息中准确提供目标、范围、权限、依赖和必要取舍, 因此大多数子任务不需要继承主线程对话。

## 决策
- 采用: 每次新建子代理前都根据当前工具能力确定最小充分的历史范围; 可控时显式配置, 默认使用不继承主线程对话的干净上下文, 由结构化任务消息和子代理可访问的稳定材料提供必要信息。
- 采用: 只有最近对话包含无法安全压缩的必要用户意图或已确认取舍时, 才选择工具支持的最小有限历史范围; 工具不支持有限继承时, 优先使用干净上下文并把必要信息提炼到任务消息; 工具不能控制历史时, 避免在消息中重复已继承内容。只有完整对话本身是不可替代的任务输入时, 才主动选择全量继承。
- 采用: 具体字段名、合法值和配置方式由当前工具接口承接, skill 只维护不继承、有限继承和全量继承的选择语义。
- 采用: 无论继承多少历史, 子代理任务仍显式定义目标、范围、权限、依赖、必要上下文、约束、验收和输出; 同一子任务需要继续处理且工具支持延续原子代理时, 继续使用原子代理。
- 不采用: 依赖工具默认值、在可复用 skill 中绑定单一工具字段, 或为了省略任务定义而继承完整历史。
