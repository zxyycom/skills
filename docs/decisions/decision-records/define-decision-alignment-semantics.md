## 目的
- 区分决策是否仍然有效，以及当前实现、行为 owner 和事实是否已经达到决策要求的状态。
- 让面向未来的决策从确认时起就能约束后续工作，同时如实表达当前实现尚未完成应用的事实。
- 防止把未对齐状态误解为决策尚未生效、实施已经开始、可以自由偏离，或者可以用进度百分比替代的任务状态。

## 背景
- 决策可以直接描述已经采用的当前基线，也可以先确定未来必须达到的目标；后一类决策在确认后已经生效，但实现可能尚未开始，或尚未推进到目标状态。
- `active` 和 `archived` 只表达生命周期，无法区分活动决策已经完全落实，还是仍存在明确的实现差距。
- “过渡中”容易暗示实施已经开始；“未对齐”只陈述决策目标与当前实现之间仍有差距，更准确地覆盖尚未启动和正在推进两种情况。
- 未对齐本身不应成为任意设计或操作的许可；只有决策明确留下的边界内才存在可选择空间。

## 决策
- 采用: 决策同时表达生命周期和对齐状态；活动决策的对齐状态只能是 `aligned` 或 `unaligned`，归档决策的对齐状态为 `null`。
- 采用: `aligned` 表示当前实现、行为 owner 和相关事实已经达到决策要求，活动决策构成必须完整遵守的当前基线。
- 采用: `unaligned` 表示决策已经生效，但当前实现、行为 owner 或相关事实尚未达到决策要求；它不要求实施已经开始，也不表示决策可以被忽略。
- 采用: `unaligned` 不自动授予实现、操作或抽象空间；只有未对齐说明中明确写入过渡边界的内容可以暂时保留差异或选择。
- 采用: 活动且未对齐的决策必须包含 `未对齐说明`，并分别写清当前差距、过渡边界和对齐条件；目标状态继续由完整 `决策` 章节表达。
- 采用: 已标记为 `aligned` 的决策发生事实偏离时，将其视为一致性问题，不通过把原记录改回 `unaligned` 来放宽约束；新的未来方向通过新的决策表达。
- 采用: 对齐状态是决策与当前事实之间的离散关系，不记录任务列表、完成百分比或执行日志。
