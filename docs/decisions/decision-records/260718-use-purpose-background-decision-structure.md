# 使用目的、背景和决策的三层投影

## 索引摘要
- 目的: 让当前决策显式回答希望达成什么、为何需要选择以及最终选择什么，同时保持低成本检索和可靠回放。
- 背景: 只在摘要增加目的会让摘要承载正文未显式出现的语义；继续由背景隐含目的又要求未来维护者自行推断，而完整目的本身可能需要展开。
- 决策: Markdown 使用索引摘要、目的、背景和决策四个必需章节；schema v2 索引保存摘要投影，既有生命周期与关系事务保持不变。

## 目的
- 让未来维护者无需从背景反推，即可明确一项决策希望长期达成或维护的结果。
- 让完整章节可以按需展开，让摘要和 JSON 保持低成本读取，并明确三者之间的 owner 关系。

## 背景
- 原有正文和索引只显式区分背景与决策，目的通常混在背景中或依赖读者自行推断。
- 目的回答希望达成或维护什么，背景回答为何需要作出选择；两者相关但不能稳定互相替代。
- 只在 `索引摘要` 和 JSON 中增加目的，会让摘要不再是正文的压缩投影，而成为承载独有语义的正文。
- 完整目的可能需要说明对象、范围或优先级；即使简单时只有一句话，也需要在正文中显式存在。
- JSON 当前索引仍应只承担当前成员和低成本检索，不复制完整决策内容或成为另一份语义 owner。
- 现有稳定路径、独立逻辑归档和真实演进才声明关系的规则不受本次字段调整影响。

## 决策
- 采用: 决策 Markdown 固定使用 `索引摘要`、`目的`、`背景` 和 `决策` 四个必需章节，`关系` 继续按需出现。
- 采用: `目的` 写希望长期达成或维护的结果，`背景` 写促成选择的事实、问题与关键约束，`决策` 写最终采用的方向和核心理由。
- 采用: `索引摘要` 包含且仅包含目的、背景和决策三个字段；每个字段只压缩对应完整章节已经明确表达的语义，不承接正文中没有出现的独立判断。
- 采用: JSON 当前索引升级为 schema v2，每个当前条目保存 `path`、`title`、`purpose`、`background` 和 `decision`；除成员路径外，其余字段由 CLI 从 Markdown 标题和显式摘要派生。
- 采用: `docs/decisions/decision-index.json` 继续作为当前成员关系的唯一 owner；决策文件继续使用稳定路径，逻辑归档只退出当前集合，不移动或重命名 Markdown。
- 采用: 没有后续判断时允许独立逻辑归档；修订、替代、判定无效或归并真实发生时，才由新记录单向链接直接前序，`archive --by` 校验后续记录直接引用全部前序，CLI 拒绝关系环并负责反查。
- 采用: CLI 确定性检查章节、字段和 JSON 派生一致性；摘要与完整章节的语义一致性继续由 agent 与用户检查。
