# 为决策表达对齐状态

## 索引摘要
- 目的: 区分必须完全遵守的当前基线与方向已确定但仍保留过渡空间的有效决策。
- 背景: 生命周期状态只能说明决策是否仍有效，不能说明当前事实和行为是否已经与其对齐。
- 决策: 后续决策格式必须表达独立于生命周期的对齐状态；具体形式不固定，索引投影是便于筛选与校验的自然延伸。

## 目的
- 让决策的有效性与当前事实、行为 owner 是否已经对齐成为两个可独立判断的维度。
- 让人类和 agent 能够区分必须完整遵守的当前基线，以及目标方向已经确定但仍保留明确过渡空间的决策。
- 让后续格式可以筛选和机械校验对齐状态，同时不因过早固定存储形式而限制更合适的表达方式。

## 背景
- 决策有时描述当前已经采用的基线，有时决定未来应当走向的目标状态；后一类决策在确认后已经有效，但相关事实或行为可能尚未完全应用该判断。
- 仅使用 `active` 和 `archived` 等生命周期状态，只能说明决策是否仍作为后续依据，不能说明当前内容与决策之间的对齐程度。
- 把这一差异称为“是否生效”会错误地暗示尚未对齐的决策还没有约束力；把它仅视为实现进度，又无法表达仍需遵守的方向和允许保留的抽象空间。
- 已对齐与尚未完全对齐会直接改变后续工作的执行义务，因此需要成为决策格式能够稳定表达的语义；索引化则是现有决策集合为筛选和机械校验自然提供的投影方式。

## 决策
- 采用: 后续决策格式必须表达一项独立于生命周期的对齐状态，用来描述当前事实和行为 owner 与有效决策之间的关系。
- 采用: 对齐状态至少在语义上区分“已对齐”和“尚未完全对齐或处于过渡中”；具体字段名、枚举、章节和存储结构不由本决策固定。
- 采用: 活动且已对齐的决策构成完整的当前基线，相关后续工作必须完全遵守；出现差异时将其视为一致性问题，而不是自动放宽决策。
- 采用: 活动但尚未完全对齐的决策仍然有效，其目标方向和已经明确的边界继续约束后续工作；只有尚未固定或明确保留开放的部分可以保留实现、操作或抽象空间。
- 采用: 对齐状态不表示决策是否生效，也不等同于任务或实现进度；它只表达有效决策与当前事实、行为之间是否已经对齐。
- 采用: 具体格式应使对齐状态能够被可靠查询和机械校验；在存在集中索引的决策集合中，将其投影到索引以支持筛选是自然延伸，但索引结构不反过来定义对齐状态本身。
