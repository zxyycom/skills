# 使用短日期命名并允许短决策结构

## 目的
- 缩短决策文件名，同时允许简单判断保持精简、可读。

## 背景
- 决策文件名前缀使用完整日期时目录较长, 用户希望改成 `260627` 这类短日期。
- 现有正文模板要求每条决策都写背景与约束、决策过程等完整小节, 对命名、模板和维护偏好这类短决策不够友好。

## 决策
- 采用: 决策文件名使用 `YYMMDD-<status>-short-title.md`, 正文标题继续使用完整日期。
- 采用: 决策正文只强制 `## 状态`、`## 问题`、`## 决定`、`## 影响` 和 `## 验证`。
- 采用: `## 背景与约束` 和 `## 决策过程` 按需使用; 需要回放上下文、关键备选或多轮收敛时使用完整结构。
- 不采用: 为短决策省略状态关系或验证。原因是状态和验证分别承接有效性追踪与落地检查, 即使短决策也需要保留。
