# 区分编辑性修正与决策演进

## 索引摘要
- 目的: 在允许维护决策记录表述的同时，防止原地改写长期判断而丢失演进历史。
- 背景: 固定契约同时允许修改记录并要求长期修订创建后续记录，但没有明确哪些变化可以原地完成。
- 决策: 只允许不改变目的、适用范围、关键背景前提、采用方向、核心理由或关系语义的编辑性修正原地完成；任一决策语义变化都创建新记录并按真实关系切换当前成员，无法判断时按决策演进处理。

## 目的
- 允许维护者修正决策记录中的文字和链接，同时保持每项长期判断的稳定身份。
- 让会改变后续工作依据的语义变化留下可追溯的新记录、关系和当前成员变化。

## 背景
- 决策记录需要允许错别字、格式、链接和表述澄清等日常维护，否则文字缺陷只能长期保留。
- 原有契约要求修订、替代、判定无效或归并时创建新记录，但“创建或修改记录”的表述没有限定原地修改范围。
- 如果目的、适用范围、关键背景前提、采用方向、核心理由或关系语义可以原地改变，稳定路径会指向与最初确认不同的判断，Git diff 之外的关系链也无法表达这次演进。

## 决策
- 采用: 错别字、格式、链接和不改变决策语义的表述澄清可以在原稳定路径上修改。
- 采用: 原地修改后同步完整章节与索引摘要，并通过 CLI 刷新派生字段和执行结构检查。
- 采用: 目的、适用范围、关键背景前提、采用方向、核心理由或关系语义发生变化时，创建完整表达新当前判断的新记录，并按真实的修订、替代、判定无效或归并关系切换当前成员。
- 采用: 无法确认一项改动是否改变决策语义时，按决策演进处理并请求判断，不原地覆盖既有记录。
