# 2026-06-30 - 给决策记录增加状态和关系

## 状态
- 当前状态: amended
- 导致状态变化的决策: [2026-06-30 - 使用短日期命名并允许短决策结构](260630-active-use-compact-decision-records.md)
- 状态说明: 状态值、状态关系和影响面目录规则仍有效; 文件日期前缀和正文结构以后续决策为准。

## 问题
- 决策记录会随项目演进过期、被替代或被判定与后续规则冲突。
- 旧记录如果仍按原文件名和索引展示, 后续维护者容易把历史判断误读成当前仍完整有效的规则。

## 背景与约束
- 决策目录已经按影响面 ID 组织, 影响面仍是后续查找决策的主要入口。
- 决策记录不是变更日志, 旧记录只有在能解释历史原因或当前结构来源时才值得保留。
- 用户明确希望状态能体现在决策目录或文件名中, 并能标明是哪条后续决策导致旧决策过期或冲突无效。

## 决策过程
1. 先评估只在正文写状态的方案: 结构最稳定, 但目录浏览时仍无法看出旧记录是否当前有效。
2. 再评估按状态建目录的方案: 状态可见, 但会把影响面目录降为二级查找, 与现有“影响面是稳定责任边界”的规则冲突。
3. 因此选择在文件名中加入状态段, 保持影响面目录不变, 同时让文件树能直接显示当前状态。
4. 状态变化原因不能只写自然语言说明, 需要在旧记录 `## 状态` 中链接导致变化的后续决策。

## 决定
- 采用: 决策文件名包含状态段, 状态值为 `active`、`amended`、`superseded` 或 `invalidated`。
- 采用: 每个决策文件在标题后增加 `## 状态`, 写明当前状态、导致状态变化的决策和状态说明。
- 采用: 非 `active` 决策必须链接导致状态变化的后续决策; `active` 决策写“无”。
- 采用: 影响面目录继续作为一级目录, 不按状态新建顶层目录或二级状态目录。
- 不采用: 只在索引中标注状态; 原因是单独打开文件或浏览目录时仍可能误读。
- 不采用: 按状态目录移动决策; 原因是状态会变化, 而影响面是更稳定的查找边界。
- 触发条件: 后续任何决策替代、修订或判定旧决策无效时, 必须同步更新旧记录状态、文件名和索引链接。

## 影响
- 决策目录的文件名可以直接反映当前有效性。
- 历史决策仍能保留上下文, 但不会被误认为当前完整有效。
- 决策校验需要检查文件名状态、正文状态和非 `active` 决策的来源链接。
