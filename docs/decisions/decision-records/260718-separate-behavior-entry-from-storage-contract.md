# 2026-07-18 - 分离行为入口与精确存储契约

## 索引摘要
- 目的: 让日常决策恢复只加载必要语义，同时让精确维护继续遵循唯一、完整且可校验的固定契约。
- 背景: 目的、背景和决策已经足以理解当前判断；若在 skill 入口再次概括正文结构、关系和事务规则，会形成契约的压缩副本，而 CLI 又必须同时服务只读恢复和写入维护。
- 决策: `SKILL.md` 承接触发、语义恢复、偏离判断、候选确认、CLI 编排和交付，并只保留目的、背景、决策的阅读语义；随包 reference 作为唯一精确存储契约，在写入、集合变更、修复或结构审阅前完整读取。

## 目的
- 让只读恢复、偏离分类和未确认候选只加载判断所需的语义与操作指引，降低日常触发成本。
- 让目录、索引、正文、关系和维护事务继续只有一个精确 owner，避免入口摘要与固定契约长期漂移。

## 背景
- `decision-records` 会在长期决定形成或与既有决定冲突时主动触发，许多调用只需要恢复当前判断，并不会进入写入或结构维护。
- JSON 索引和 Markdown 已经用目的、背景和决策明确表达希望达成什么、为何需要选择以及最终采用什么；这些语义足以定位、理解和判断相关决策。
- 在入口补充简化的决策结构说明会重复固定契约，且无法替代精确维护时对完整格式、关系和事务约束的读取。
- CLI 既承担 `list`、`trace` 和 `check` 等只读操作，也承担当前成员变更和派生字段同步，不能整体下沉到精确契约。
- 现有随包 reference 作为不同项目共享的唯一固定契约、目标项目不复制契约以及项目专属更高门槛留在行为 owner 的方向继续有效。

## 决策
- 采用: `SKILL.md` 承接触发、语义恢复、偏离判断、候选确认、CLI 调用时机、维护流程编排和交付标准。
- 采用: `skills/decision-records/references/decision-record-rules.md` 继续作为唯一随包固定契约，独立承接目录、索引 schema、稳定身份、正文格式、关系、生命周期、维护事务和结构校验语义。
- 采用: 只读恢复、偏离分类和未确认候选不读取固定契约；创建、修订、逻辑归档、归并、显式修复或结构审阅前完整读取。
- 采用: 入口只说明如何从目的、背景和决策恢复当前判断；简化模板、字段约束和正文结构统一由固定契约承接。
- 采用: CLI 继续保留在入口中，作为查询、关系追溯、当前成员变更、派生字段同步和结构检查的确定性接口；固定契约说明各命令必须满足的精确约束。
- 采用: 目标项目不复制固定契约；项目专属的更高记录门槛继续写入 `AGENTS.md` 或相关行为 owner。
