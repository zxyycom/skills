# 查询 Markdown 投影的决策元数据

## 索引摘要
- 目的: 让日常恢复按主题、生命周期和对齐关系筛选，同时保持 Markdown 是全部状态元数据的唯一事实源。
- 背景: 查询需要集中索引，但把状态和时间描述为索引独有元数据会与索引可重建契约冲突。
- 决策: list、show 和 trace 查询 JSON 投影；状态、alignment 与创建时间均来自 Markdown，索引不拥有独立值。

## 目的
- 让日常任务默认只恢复仍作为后续依据的活动决策，并能按主题、生命周期和对齐关系缩小结果。
- 让 `list`、`show` 和 `trace` 分别承接集合筛选、单条展开和关系回放，避免用全量列表替代逐级查询。
- 让查询效率不改变 Markdown 事实源与派生索引的 owner 分工。

## 背景
- 默认查询归档或全部记录会把历史噪声带入日常判断，因此状态默认值应继续是 `active`。
- 主题已经是决策路径的第一段稳定分类，可以直接作为筛选键，不需要增加另一份标签或缓存。
- 列表中的标题、目的、背景和决策足以定位相关记录；关系和完整正文继续按需展开。
- JSON 索引适合集中筛选，但其中的生命周期、对齐状态和创建时间都来自 Markdown frontmatter。

## 决策
- 采用: `list` 默认筛选 `active`，使用 `--status active|archived|all` 显式改变生命周期范围。
- 采用: `list --topic <topic-id>` 按相对路径第一段筛选 kebab-case 主题，并可与 `--status`、`--alignment` 和 `--full-time` 组合；合法但没有记录的主题返回空结果和退出码 `0`。
- 采用: 归档或全部状态用于历史恢复、结构修复或全库生命周期审计；对齐筛选只读取索引对 Markdown alignment 字段的投影。
- 采用: `list` 输出生命周期、对齐状态、创建日期或秒级时间、路径、标题、目的、背景和决策，不输出关系。
- 采用: `show <path>` 输出路径，以及从 Markdown frontmatter 投影的生命周期、对齐状态和秒级创建时间，再输出原始 Markdown。
- 采用: `trace <path>` 使用索引中的直接关系构建前序和后续关系图，支持方向和非负最大深度，并显示各节点的生命周期与对齐状态。
- 采用: 查询结果写入标准输出，warning 写入标准错误并限制结论完整性；查询输出不成为任何状态的独立 owner。
