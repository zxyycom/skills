# 2026-06-27 - 补充决策记录目的说明

问题:
- 决策记录已经有记录范围和写法, 但对“为什么要记录决策”的说明偏薄。
- 如果目的不清楚, 后续维护者可能把它当成普通变更日志, 记录过多执行细节；也可能相反, 只留下结果而丢失关键取舍。
- 这个项目本身是 skill 维护项目, 修改过程中的判断方式有复用价值, 需要被明确说明。

决策过程:
- README 负责项目介绍, 不适合展开维护思考方法。
- AGENTS 只保留稳定项目约定, 不适合承接决策记录的细节说明。
- `docs/decisions/` 本身是决策记录 owner, 最适合说明记录目的、使用价值和边界。

决定:
- 在决策记录索引开头增加“为什么记录决策”小节。
- 明确决策记录用于保留取舍依据、区分历史材料和当前规则、降低反复讨论成本、支持安全演进和帮助 agent 稳定协作。
- 同时说明决策记录不是完整变更日志, 普通文本修正和一次性执行细节不进入本文档。

影响:
- 后续记录决策时, 应优先写可复用判断, 少写流水账。
- 后续阅读决策记录时, 可以把它作为理解项目长期偏好的入口。
- 决策记录本身的 owner 更清晰, 不需要把这类说明散落到 README 或 AGENTS。

验证:
- 决策记录索引开头已补充“为什么记录决策”小节。
- 记录范围和写法仍由决策记录目录承接。
