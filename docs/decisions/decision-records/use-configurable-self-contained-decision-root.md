# 使用可配置且自包含的决策根目录

## 索引摘要
- 目的: 让默认位置和显式目标路径共享稳定身份，并保持 Markdown 事实源与派生索引的自包含契约。
- 背景: 决策目录既可以使用默认位置，也可以指向 root 外部；若混淆目标选择与路径解析，会让同一参数落到不同集合。
- 决策: 默认使用 root/docs/decisions；显式 decisions-dir 直接选择目标，绝对路径可在 root 外，非绝对路径只相对 root 解析。

## 目的
- 让 CLI 的目录覆盖能力具备明确、可移植且不会分裂身份的契约。
- 保持默认项目零配置，同时允许调用者显式选择 `--root` 外部的决策目录。
- 让任意有效根目录都遵循 Markdown 唯一事实源和索引可无损重建的同一模型。

## 背景
- 默认布局长期使用 `docs/decisions/decision-index.json` 和主题目录，能够满足大多数项目。
- `--root` 已经是默认位置和相对参数的稳定解析基准；当前工作目录只在没有显式 `--root` 时提供该参数的默认值。
- `--decisions-dir <path>` 是显式目标选择，不是“必须位于 root 下”的目录片段；绝对路径可能指向另一个位置和 Git 仓库。
- 根目录选择只应改变存储位置，不应改变 Markdown、生命周期、关系或查询投影的 owner 分工。
- 每条当前格式 Markdown 已经自包含生命周期、对齐状态、创建时间、语义和直接关系，因此索引缺失不再造成成员身份无法恢复。

## 决策
- 采用: 将保存主题 Markdown 和 `decision-index.json` 的解析后目录定义为决策根目录；该目录必须位于能够读取 `HEAD` 的 Git 仓库，但不要求位于 `--root` 内。
- 采用: 未传 `--decisions-dir` 时使用 `<root>/docs/decisions`；传入绝对路径时直接使用该目标，传入非绝对路径时只相对 `--root` 解析。
- 采用: `--root` 默认是当前工作目录，但显式 `--decisions-dir` 不再相对当前调用位置、决策文件或 Git 根目录二次解释。
- 采用: 对同一集合的全部查询和维护使用同一个解析后的决策根目录，不能在调用之间改变解析基准。
- 采用: 记录的稳定身份是相对根目录的 Markdown 路径，根目录本身不进入身份；目录覆盖不产生第二套格式或状态语义。
- 采用: 每条 Markdown 承接自身全部权威状态与语义，`decision-index.json` 只承接从根目录内全部有效 Markdown 确定性生成的全生命周期查询投影。
- 采用: 根目录整体尚不存在时把集合解释为尚未初始化且查询不创建文件；已有当前格式 Markdown 但索引缺失或损坏时，从 Markdown 完整重建索引。
- 采用: Markdown 自身无效时修复对应事实源或请求无法恢复的判断，不从索引或默认值反向制造状态。
