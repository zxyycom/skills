# 让行为入口聚焦任务路由

## 索引摘要
- 目的: 让频繁触发的 skill 入口只加载恢复和决策所需行为，同时保持精确维护遵循唯一固定契约。
- 背景: 行为入口已经与存储契约分离，但仍重复自动初始化条件、编辑边界、CLI 参数和事务细节，形成了容易漂移的压缩契约。
- 决策: `SKILL.md` 只承接内容 owner、主动读取、任务出口、恢复、候选、命令选择和分出口验收；目录、字段、关系、维护事务与 CLI 精确语义继续只由固定契约完整承接，入口在精确维护前加载该契约。

## 目的
- 降低长期决策触发时的入口负担，让 agent 先获得足以选择下一步的行为规则。
- 让存储和事务约束只有一个完整 owner，减少入口摘要与固定契约之间的重复和漂移。
- 保留查询 warning 的行为解释、命令选择和交付标准，因为这些内容直接影响 agent 的执行路径。

## 背景
- 既有判断已经将 `SKILL.md` 定位为行为入口，将随包 reference 定位为唯一精确存储契约。
- 入口后续又加入了首次激活的精确条件、编辑性修正字段边界、完整命令变体和失败语义，这些内容在固定契约中已有完整版本。
- 日常恢复主要需要知道从哪里读取、如何判断相关性、何时追溯、如何处理 warning 和何时进入维护，不需要提前加载全部存储约束。
- 创建、修订、归档、迁移和结构审阅仍必须完整读取固定契约，因此收紧入口不会降低维护准确性。

## 决策
- 采用: `SKILL.md` 承接触发后的内容 owner、主动读取、任务出口、语义恢复、偏离分类、候选确认、CLI 选择和按出口交付。
- 采用: 固定契约独立承接决策根目录、索引 schema、稳定身份、正文格式、关系、生命周期、维护事务、CLI 合法值和失败恢复。
- 采用: 入口只保留能够改变执行路径的 CLI 说明和核心调用形式，精确参数及事务条件引用固定契约与 `--help`。
- 采用: 恢复、偏离分类和未确认候选不加载固定契约；创建、修订、逻辑归档、归并、显式修复或结构审阅前完整读取。
- 采用: 面向人类的 skill 说明解释核心体验和 owner 分工，不复制完整命令与事务规则。
- 采用: 目标项目不复制固定契约；项目专属的更高记录门槛继续写入 `AGENTS.md` 或相关行为 owner。
