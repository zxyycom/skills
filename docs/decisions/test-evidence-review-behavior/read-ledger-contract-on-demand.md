# 按任务出口渐进读取测试账本契约

## 索引摘要
- 目的: 让测试价值审查先恢复相关契约和证据，只在精确维护需要时加载完整账本格式，降低无关上下文成本。
- 背景: 固定契约包含大量字段、配置和 CLI 细节，纯语义审查每次完整读取会挤占实现、测试和行为 owner 的有效上下文。
- 决策: 所有任务继续受账本契约约束，但仅在写入、结构审阅、修复或精确诊断时完整读取；日常恢复先用 list/show/check。

## 目的
- 让 agent 在测试任务中优先理解当前行为 owner、实现、测试和已有证据，不让未使用的存储细节遮蔽语义判断。
- 保证任何账本写入、初始化、结构修复和精确字段解释仍以完整固定契约为前置条件。
- 让 CLI 或运行时不可用时的降级路径明确区分可完成的语义审查与尚未机械验证的映射、Scope 和 trigger。

## 背景
- `ledger-contract.md` 同时承接 automated、review、exempt 字段组合、marker、Git Scope、配置、CLI 和机器接口；多数纯测试价值审查只消费其中少量概念。
- 既有 skill 要求每次在读取项目事实后立即完整读取固定契约，导致只读审查也承担与写入任务相同的上下文成本。
- 新增 `list`、`show` 和 inspection 后，agent 可以先恢复相关 case、源码映射和 trigger，再决定是否进入精确维护。

## 决策
- 采用: 每次调用 `test-evidence-review` 都继续受固定账本契约约束，已有账本仍必须进入机械对账；渐进读取只改变何时加载完整引用，不改变登记义务和字段语义。
- 采用: 日常恢复先读取项目规则与行为 owner，运行 `list` 定位 case、按需 `show` 展开，再运行严格 `check` 取得诊断。
- 采用: 初始化或写入账本、修改入口角色、修复结构或映射、审阅账本契约，以及处理需要精确字段语义的诊断前，完整读取 `ledger-contract.md`。
- 采用: CLI 或 Node 不可用时直接读取现有账本和源码完成能够证明的语义审查，并披露未机械验证范围；无法恢复 Git 状态的 Scope 检查返回 blocked。
- 不采用: 因为暂未完整读取固定契约而跳过账本、入口映射或 review trigger，也不把查询成功解释为严格校验通过。
