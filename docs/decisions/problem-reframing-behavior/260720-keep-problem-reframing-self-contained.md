## 目的
- 让 agent 在继续优化局部解法前, 判断当前问题是否值得并适合按现有方式解决。
- 让问题重构产出能够直接进入后续分析、设计或实现的问题表述、依据和下一步动作。

## 背景
- Agent 容易从已有接口、功能、流程或实现限制开始推理, 并把当前解法隐含的问题表述当作既定目标。
- 局部规则、例外和补偿机制持续增加时, 更细致地优化当前路径可能只会固化错误前提或错误层级。
- 问题重构同时适用于工程、产品、流程、研究、提示词和协作场景, 因此需要围绕通用的目标、证据、前提、边界和抽象层工作。
- 放大视野本身不是目标; 没有范围约束的问题重构会把局部优化替换成宏大但不可执行的分析。

## 决策
- 采用: 将 `problem-reframing` 作为独立的流程与决策型 skill, 由用户明确请求或可观察的框架风险触发。
- 采用: 先恢复真实结果、事实、约束、前提、问题边界、抽象层和当前解法, 再只沿可能改变选择的方向形成候选。
- 采用: 允许保留、缩小、改写、上移或放弃当前问题; 选择能够解释真实结果、控制原因且范围最小的框架, 证据不足时先做最小调查。
- 采用: 将选定方向、问题表述、关键依据、约束、未知项和下一步动作交回当前任务; 后续领域工作继续服从对应行为 owner 和用户授权。
