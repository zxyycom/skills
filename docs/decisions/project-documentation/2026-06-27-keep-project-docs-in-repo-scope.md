# 2026-06-27 - 项目文档只保留仓库内信息

问题:
- 长期项目文档如果写入仓库外路径、外部链接、工具安装位置或一次性调查来源, 会让项目介绍变成环境快照。
- 这类信息会降低文档可迁移性, 也会让后续维护者误以为外部位置是本项目依赖。
- 用户明确要求文档重点放在项目基础介绍和决策记录, 不沉淀超出本项目的地址或特定外部流程词。

决策过程:
- README 的实际用途是项目入口说明, 应只回答“这个项目是什么、文件如何组织、维护时看哪里”。
- AGENTS 的实际用途是 agent 维护约定, 应描述本仓库内的读取和编辑边界, 不保存外部来源。
- 决策记录的实际用途是沉淀长期判断, 可以记录“不要写入仓库外信息”这个规则, 但不记录具体外部地址。

决定:
- README 增加项目定位和 skill 内容结构, 强调本仓库是 skill 维护项目。
- AGENTS 删除仓库外路径和外部流程参考, 改成通用规则: 项目文档只写本仓库内可维护的信息。
- 决策记录新增本条决策, 后续遇到类似信息时只记录抽象判断, 不写入具体外部地址。

影响:
- 后续项目文档应只引用本仓库内路径。
- 仓库外上下文可以用于本次判断, 但只作为临时输入, 不进入长期文档。
- 需要说明外部来源时, 放在对话、提交说明或一次性调查记录中, 不放入 README、AGENTS 或决策记录。

验证:
- README、AGENTS 和决策文件不包含仓库外路径或外部链接。
- README 的项目介绍覆盖项目定位、目录结构、skill 内容和维护约定。
- 决策记录包含记录范围、写法和本次文档收敛决策。
