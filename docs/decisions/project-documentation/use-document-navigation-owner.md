# 使用仓库导航统一任务路由与内容 owner

## 索引摘要
- 目的: 让维护者和 agent 按任务快速定位先读内容及其 owner，避免重复搜索和规则归属漂移。
- 背景: 项目已有多类稳定内容和 owner，但读取入口分散在 README、AGENTS、docs 与 skill 目录中。
- 决策: 使用 docs/navigation.md 统一维护任务路由、内容 owner 和仓库位置，其他入口只保留摘要和链接。

## 目的

- 为人类和 agent 提供一个稳定入口，能够按任务找到必读文档、内容 owner 和需要时再读的材料。
- 保持规则完整解释只存在于对应 owner，避免导航入口成为第二套规范。

## 背景

- 本仓库已经分别使用 `README.md`、`AGENTS.md`、`docs/repository-model.md`、工程文档、skill 本体、人类介绍和决策记录承接不同内容。
- 这些 owner 的分工已经存在，但任务入口分散；新增 skill 或项目文档后，维护者需要从目录结构和多份入口说明中自行恢复读取路径。
- 按任务路由到主 owner 的仓库导航模式能够集中读取入口，同时让规则细节继续留在各自文档。

## 决策

- 采用: 新增 `docs/navigation.md`，使用任务路由和内容 owner 两种视角说明先读什么、需要时再读什么，以及稳定内容类型位于何处。
- 采用: `docs/navigation.md` 只保存路由、路径模式和维护规则，不复制 skill 行为、仓库模型、工具链、编码规范或决策契约。
- 采用: 由 `AGENTS.md` 和 `README.md` 提供仓库导航入口，并在新增、移动或移除稳定内容类型及其 owner 时同步更新导航。
- 采用: 单个 skill、单条决策和实现文件继续通过稳定路径模式发现，不在导航中维护逐项清单。
