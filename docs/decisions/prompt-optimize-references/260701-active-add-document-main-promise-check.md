# 2026-07-01 - 给 prompt-optimize 增加文档主承诺检查

## 状态
- 当前状态: active
- 导致状态变化的决策: 无
- 状态说明: 作为当前 `prompt-optimize` 判断文档重心偏移的执行流程使用; 本决策扩展 2026-06-30 默认执行路径, 不改变该 skill 的引用文件 owner 分工。

## 问题
- `prompt-optimize` 已经要求判断用途、阅读脉络和内容 owner, 但 owner 主要解决内容完整解释放在哪里, 不足以稳定识别整篇文档的重心是否偏离入口信号建立的预期。
- 用户反馈指出, 仅看标题、部分序言或其他入口信号就能推断文档基调时, 后续正文重心却落到其他主题, 应被视为明显偏离; SOP、runbook、API、字段说明和规则索引同样可能被背景、例外、历史或实现细节带偏。

## 背景与约束
- 当前 `prompt-optimize/skill/prompt-optimize/SKILL.md` 是该 skill 默认执行路径的完整 owner。
- `principles.md` 负责解释长期理由; `workflows.md` 和 `rewrite-rules.md` 是迁移期保留副本, 不参与默认读取。

## 决策过程
1. 先识别到现有“用途”和“owner”规则虽然能处理内容归位, 但没有明确要求从文档入口信号建立整篇文档的主承诺。
2. 再把问题拆成两个判断: owner 判断“内容完整解释应放在哪里”, 重心判断“当前文档是否兑现入口信号建立的主承诺”。
3. 最终决定把主承诺检查放在“定任务出口”之后、“选择文档骨架”之前, 让后续骨架、行为审计和 owner 判断都有同一个基准。

## 决定
- 采用: 在 `prompt-optimize` 主执行流程中新增“建立文档主承诺”, 从标题、副标题、序言、开头段、一级标题、触发条件、前置条件、资源名、字段集合、适用范围或输入输出描述推断核心问题、中心对象、预期展开方向和读者期待的结论或操作结果。
- 采用: 在行为审计中新增“重心偏移”类型, 用入口承诺与正文主题、入口问题与后文落点、背景/例外/历史/实现细节/示例占比来识别偏离。
- 采用: 在 owner 步骤中明确区分 owner 判断和重心判断; 当前节点即使是 owner, 若内容比例压过主承诺, 也需要降级、移动、删减或收窄入口承诺。
- 采用: 在完成检查中加入文档主承诺是否被正文层级、篇幅、结论、验收或校验方式兑现。
- 采用: 在 `principles.md` 增加“文档主承诺原则”, 解释为什么主承诺检查先于 owner 判断。
- 不采用: 只强化 owner 规则。原因是 owner 只能定位内容归属, 不能单独比较入口承诺与全文实际重心。
- 不采用: 同步改写 `workflows.md` 和 `rewrite-rules.md`。原因是它们是迁移期保留副本, 当前默认执行路径不再以它们为 owner。
- 触发条件: 后续维护 `prompt-optimize` 涉及文档重心、阅读脉络、入口承诺、正文篇幅比例或 owner 归位时, 先沿用主承诺检查。

## 影响
- `prompt-optimize` 审阅或改写文档时, 会先形成文档主承诺, 再判断正文重心是否兑现该承诺。
- 后续维护该 skill 时, 与文档重心相关的执行流程优先更新入口文件; 原理解释更新 `principles.md`; 迁移期保留副本不因普通规则增强而重新成为默认 owner。
