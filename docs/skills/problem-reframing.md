# Problem Reframing

Agent 很容易把当前解法当成问题本身: 接口已经存在就继续完善接口, 功能已经开始实现就继续补齐分支, 流程已经运行就继续降低每一步的成本。局部方案可能越来越完整, 真实结果却没有改善, 因为目标、前提、问题边界或抽象层从未被重新检查。

`problem-reframing` 是一项流程与决策能力。它在出现框架风险时暂停当前解法, 恢复真实消费者、场景、期望结果、事实、约束和未验证前提, 再检查问题的必要性、边界、因果位置与抽象层。它不默认追求更大的方案, 而是寻找能够解释真实结果的最小框架。

结果可以是保留原问题、缩小范围、按真实结果改写问题、上移到能够控制原因的层级, 或放弃缺少价值的工作。证据不足时, 先选择能够区分这些方向的最小调查、实验或度量。

这个能力适用于工程、产品、流程、研究、提示词和协作等场景。它把选定方向、重新表述的问题、关键依据、约束、未知项和下一步动作交回当前任务; 后续领域分析、实现和验证仍由对应行为 owner 承接。
