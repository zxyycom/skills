# Investigation Report

`investigation-report` 用于让 agent 稳定形成可快速汇报、降低上手门槛并支持接续探索的调查报告。它不是把搜索过程原样保存下来，而是把每轮调查整理成带形成时间的阶段性认识：为什么调查、怎样取得证据、得到了什么结果、能够推断到哪里、还不知道什么，以及下一位调查者从哪里继续。

## 什么时候使用

本 skill 采用非主动式入口。用户明确要求记录、沉淀、创建、更新或审阅调查报告时使用；普通调查、排障和问答不会因为结果看起来有参考价值就主动建议建报告。

常见报告对象包括：为了实现某项能力调查候选库，研究同类产品或竞品，深入分析机制与性能，恢复系统的实际执行逻辑，以及研究并发、时序和状态协作。它们只是帮助识别适用内容，仍以用户明确要求沉淀为前提。

## 报告怎样支持汇报和接续

报告顶部的固定概述始终投影当前状态。`当前认识` 先给出核心问题目前最直接的答案，再说明适用边界和影响理解的主要未知；读者不展开全文，也应能知道为什么调查、调查到哪里以及是否继续。

正文使用一个固定的 `调查记录` 容器，每轮形成阶段性认识时增加一个调查段。调查段只固定语义标题和形成时间；形成时间之后先写本轮可直接复述的结果及其边界，不能回答时先说明缺少什么证据，再根据问题展开背景、范围与方法、证据、推断、冲突、未知和接续方向。这样旧段保留当时背景，新段明确既有认识被加强、削弱、推翻还是限缩，顶部概述则始终提供最新汇报口径。

每轮调查不是一次命令或一个来源，而是能够独立说明一段认识的工作单元。报告为没有参与原对话的读者写作，围绕“主张—证据—解释—边界”组织调查段，不把搜索、试错和原始日志直接堆成正文。

## 怎样形成高质量调查段

重要事实需要保留来源、方法、时间和条件，并说明它能够和不能证明什么。事实、基于证据的推断、有限范围内未观察到的现象和未知应明确区分；提出原因时还要区分相关性、候选机制和已经覆盖关键环节的因果证明。

不同对象可以采用不同表达方式：库调研强调目标、约束、比较口径和适用边界；竞品调研强调观察时间、事实来源、机制差异和参考边界；性能分析强调负载、基线、测量和因果强度；实际执行逻辑与并发研究强调入口、调用或状态链、关键分支、时序和同步关系。

完成一轮后先写调查段，再刷新顶部当前认识和最近更新时间。报告第一次形成可用调查段的时刻记录为首次形成时间；每个调查段另有自己的形成时间，实验、日志和外部来源仍保留各自观测时间。

## 整理与检查

报告按主题语义路径保存，并通过索引提供发现入口。固定概述、调查记录外壳、状态和时间用于稳定整理；调查段内部仍由问题决定。

随 skill 分发的 `check-investigations.mjs` 默认检查调查索引和全部报告，也可以按主题或报告路径筛选。它只验证目录、固定概述、调查段形成时间和索引投影等机械结构，不判断报告是否便于汇报、证据是否可靠或分析是否正确。
