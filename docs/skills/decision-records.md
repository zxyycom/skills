# Decision Records

`decision-records` 用稳定 Markdown 文件保存重要长期判断的完整目的、背景、决策和按需声明的演进关系，用版本化 JSON 全生命周期索引保存全部记录的状态、秒级创建时间、精简摘要和直接关系。它关注“希望长期达成什么、为何采用当前方向、以后怎样不偏离”，不承接通用知识、任务日志或当前实现状态。

## 核心体验

日常工作继续以原任务为主。当用户正在形成可能持续影响后续行为、owner、边界、兼容性、风险处理或验收方式的决定，或拟议决定可能与既有决定冲突时，agent 会在确定方向或实施依赖该方向的实质改动前恢复相关活动判断。

恢复默认从活动记录开始，`list` 提供足以定位判断的基础元数据，主题明确时可以进一步筛选。需要完整理由时运行 `show`，需要审计演进时运行 `trace`；只有任务明确涉及历史或全生命周期结果时才扩大查询范围。

索引登记活动和归档记录，并保存创建时间、精简摘要和直接关系。它既支持日常筛选，也能在需要时恢复先后与演进；关系通过 `trace` 展开，不挤占列表结果。CLI 或 Node 环境不可用时，索引、Markdown、固定契约和随包源码材料也能支持工具恢复或索引重建。

决策集合尚未初始化时不创建空索引；首条长期判断确认后再进入正常维护。已有材料但索引或运行时不可用，以及旧集合需要升级时，agent 才读取维护恢复手册。手册提供替代运行时、随包更新器、GitHub release 下载、源码重建、最小 CLI 复现和直接索引维护路径。

发现当前任务与既有判断不一致时，agent 会区分对齐、明确的一次性例外、长期修订和一致性问题。只改变本次任务执行方式的指令属于一次性例外；只有用户已经决定改变后续工作的默认依据时才属于长期修订。

当前指令已经明确长期判断、维护对象和影响范围时，agent 会直接执行而不重复询问；新增记录或改变生命周期前，会明确告知用户将写入或改变的判断及状态变化。由 agent 提炼的新候选或范围不清的调整仍需确认。

原地修改只承接不改变决策语义的编辑性修正。决策语义变化时，agent 会创建新记录并保留真实演进关系；具体格式、生命周期事务和字段限制由固定契约统一定义。

查询带 warning 返回时，索引中仍可恢复的结果会继续输出，但不能据此断言缺失的正文或关系不存在。agent 会说明不完整性及其对结论的影响，写入完成仍以严格 `check` 为准。

## 设计边界

1. 可审核：生命周期状态、索引摘要、Markdown 内容和历史关系都能进入版本审查。
2. 语义分层：完整章节承接完整语义，Markdown 摘要用于检索，JSON 提供集中索引。
3. 可演进：语义变化保留新的完整判断和直接关系，普通退役可以独立归档。
4. 路径稳定：生命周期、关系或标题润色不改变既有身份。
5. 防偏离：长期决策形成或发生冲突时恢复活动判断，明确区分一次性例外与长期修订。
6. 查询可用：中央索引可解析时，局部问题以 warning 暴露，不阻断其他可恢复结果；warning 同时限定结论完整性。
7. 范围可控：查询、候选和维护分别按自己的任务出口验收，不把查询自动扩大为修复。

## Owner 分工

1. [`SKILL.md`](../../skills/decision-records/SKILL.md) 承接 agent 的发现、语义恢复、偏离判断、候选确认、调用方式选择和按任务出口交付。
2. [`decision-record-rules.md`](../../skills/decision-records/references/decision-record-rules.md) 是目录、索引 schema、稳定身份、正文格式、关系、生命周期、维护事务和结构校验语义的唯一固定契约。
3. [`decision-index.schema.json`](../../skills/decision-records/references/decision-index.schema.json) 随 skill 分发，向编辑器、JSON Schema 工具和临时恢复工具提供机器可读的索引结构。
4. [`maintenance-recovery.md`](../../skills/decision-records/references/maintenance-recovery.md) 只承接升级和故障恢复顺序，不重复定义固定契约。
5. `scripts/decision-records/` 承接 CLI 源码、测试和构建；随 skill 分发的 CLI 提供筛选列表、单条展开、索引关系追溯、派生字段同步和显式生命周期更新。
6. 每条 Markdown 的完整章节承接完整语义，同文件的索引摘要承接作者确认的压缩表达；决策根目录中的 `decision-index.json` 承接全部记录的生命周期元数据和派生检索投影。

## CLI 体验

1. 恢复：`list` 定位活动判断，`show` 展开单条正文，`trace` 回放关系；局部问题通过 warning 和无效标记暴露。
2. 筛选：列表可以按主题、生命周期和时间显示需求收窄结果。
3. 验证与维护：`check` 严格验证集合；其余写命令按固定契约同步索引或改变生命周期。
4. 模块复用：直接导入已安装 skill 的 `scripts/decision-records.mjs`；公开导出和类型见相邻的 `decision-records.d.mts`。

## 项目接入

目标项目使用默认决策根目录时无需额外配置；选择其他工作区内目录时，对所有 CLI 调用统一传入 `--decisions-dir`。项目 agent 指令只加入指向索引和本 skill 的短入口，项目专属记录门槛留在自己的行为 owner，不复制固定契约。
