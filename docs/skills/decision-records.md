# Decision Records

`decision-records` 用稳定 Markdown 文件保存重要长期判断的完整目的、背景、决策和按需声明的演进关系，用版本化 JSON 索引保存当前仍作为后续工作依据的精简摘要。它关注“希望长期达成什么、为何采用当前方向、以后怎样不偏离”，不承接通用知识、任务日志或当前实现状态。

## 核心体验

日常工作继续以原任务为主。当用户正在形成可能持续影响后续行为、owner、边界、兼容性、风险处理或验收方式的决定，或拟议决定可能与既有决定冲突时，agent 会在确定方向或实施依赖该方向的实质改动前恢复相关当前判断。

恢复从 `list` 和 JSON 当前索引开始，通过路径、标题、目的、背景和决策摘要定位相关记录，需要完整理由时再读 Markdown，需要审计演进时再运行 `trace`。查询带 warning 返回时，结果仍可用于恢复，但不能据此断言缺失的记录或关系不存在；agent 会说明不完整性及其对结论的影响。

决策根目录默认是 `docs/decisions`。根目录整体尚不存在表示集合尚未初始化，不需要创建空索引；首条长期判断确认后，写入 Markdown 并通过 `activate` 初始化当前集合。已有材料但索引缺失或无效时，agent 会先说明当前成员无法可靠恢复。

发现当前任务与既有判断不一致时，agent 会区分对齐、明确的一次性例外、长期修订和一致性问题。只改变本次任务执行方式的指令属于一次性例外；只有用户已经决定改变后续工作的默认依据时才属于长期修订。

当前指令已经明确长期判断、维护对象和影响范围时，agent 会直接执行而不重复询问；新增、调整或退出当前集合前，会明确告知用户将写入或改变的判断及集合变化。由 agent 提炼的新候选或范围不清的调整仍需确认。

原地修改只承接不改变决策语义的编辑性修正。决策语义变化时，agent 会创建新记录并通过真实演进关系切换当前成员，避免维护文字时覆盖形成历史。不在索引中的记录属于逻辑归档，但保留稳定相对路径并可按需查询。

入口只承接恢复、判断、候选、命令选择和交付。创建、修订、归档、迁移或结构审阅时，agent 才加载固定契约中的精确格式和事务约束。已有材料不符合契约时，先报告差异、可保留语义和修复范围，确认范围后再转换。

## 设计边界

1. 可审核：JSON 当前成员、Markdown 内容和历史关系都能进入版本审查。
2. 语义分层：完整章节承接完整语义，Markdown 摘要压缩对应章节，JSON 以路径承接当前成员并镜像摘要。
3. 可演进：语义变化通过当前成员切换和单向直接关系表达；没有后续判断的普通退役可以独立退出当前集合。
4. 路径稳定：稳定身份是相对决策根目录的路径，不因生命周期、关系或标题润色而变化。
5. 防偏离：长期决策形成或发生冲突时恢复相关判断，明确区分一次性例外与长期修订。
6. 查询可用：中央索引可解析时，局部问题以 warning 暴露，不阻断其他可恢复结果；warning 同时限定结论的完整性。
7. 范围可控：查询、候选、维护和迁移分别按自己的任务出口验收，不把查询自动扩大为修复。

## Owner 分工

1. [`SKILL.md`](../../skills/decision-records/SKILL.md) 承接 agent 的发现、语义恢复、偏离判断、候选确认、CLI 选择和按任务出口交付。
2. [`decision-record-rules.md`](../../skills/decision-records/references/decision-record-rules.md) 是目录、索引 schema、稳定身份、正文格式、关系、生命周期、维护事务和结构校验语义的唯一固定契约。
3. `scripts/decision-records/` 承接 CLI 源码、测试和构建；随 skill 分发的 CLI 提供结构检查、当前与归档查询、关系链反查、派生字段同步和校验式成员更新。
4. 每条 Markdown 的完整章节承接完整语义，同文件的索引摘要承接作者确认的压缩表达；决策根目录中的 `decision-index.json` 只承接当前成员并镜像摘要派生字段。

## CLI 体验

1. 恢复：`list` 查询当前或归档成员，`trace` 按方向和深度回放关系；局部问题通过 warning 和 `invalid` 标记暴露。
2. 验证：`check` 严格检查索引、正文、稳定路径和关系；查询成功不替代严格验证。
3. 维护：`sync-index --write`、`activate` 和 `archive` 在固定契约约束下同步派生字段或改变当前成员，失败时恢复原索引。

## 项目接入

目标项目使用默认决策根目录时无需额外配置；选择其他工作区内目录时，对所有 CLI 调用统一传入 `--decisions-dir`。项目 agent 指令只加入指向当前索引和本 skill 的短入口，项目专属记录门槛留在自己的行为 owner，不复制固定契约。
