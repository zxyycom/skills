# 真实路径行为探测

当设计发现需要确认 agent 会不会在普通请求中自然选择目标 skill、选择后如何使用，以及行为是否发生变化，并且当前环境可以提供干净的独立执行实例时，使用本协议。它只用于采集设计证据，不构成所有 skill 的固定测试门禁，也不能替代实现后的完整评估。

## 选择探测方式

1. **盲态探测**用于观察自然发现与实际采用。任务不点名目标 skill，也不提供 skill 路径、测试目的、内部诊断、预期答案或隐藏成功标准；正常运行时可见的项目规则、skill 目录和能力清单仍然保留。
2. **显式采用探测**用于区分发现失败与执行失败。在新的干净实例中明确提供目标 skill，只检查加载后是否遵循流程、引用和权限。只有盲态结果不足以判断失败环节时才补做，不能在原实例中追加提示后冒充盲态结果。
3. **对照探测**用于判断行为改善：保持现实任务、输入和环境尽量一致，对比无 skill 与有 skill、旧版与新版；只有需要支持因果或质量结论时才增加，不把普通观察升级成强制 benchmark。

目标 skill 尚未实现时，只能采集未启用它的 baseline，或设计未来探测；不得把预期的启用后行为写成已经观察到的结果。

## 设计现实任务

1. 从当前案例、失败风险和触发边界即时生成任务，不维护脱离目标领域的固定题库。至少包含应触发案例和 near miss；只有关键分支、失败恢复或人类权限会改变设计时，才增加相应案例。
2. 使用真实用户会提供的目标、材料和措辞。为了保证可执行性，可以说明范围、权限和交付，但不能把 agent 应自行作出的 skill 选择、诊断或步骤写进任务。
3. 每个案例使用一个不继承其他案例或设计讨论的新上下文；完成后的回访继续使用原实例。同一实例连续执行多个案例会产生学习和顺序影响，只能作为独立的连续使用场景，不能计作互相独立的样本。
4. 保留解释结果所需的初始条件，包括任务原文、可访问材料、可用 skill、权限、工具和相关版本或工作区状态。
5. 优先使用只读任务或隔离工作区。多个实例共享可变工作区时串行执行；写入或外部通信无法隔离，或者任务涉及高风险、不可逆操作时，不运行探测，只登记验证义务。

## 执行与回访

1. 任务开始后让实例自主完成，不在中途提示目标 skill 或成功标准。阻塞、越权请求和未完成状态也是行为证据，应按原样记录。
2. 保存可获得的执行轨迹和最终产物。探测完成后，使用原实例进行一次中性回访，并要求它不要重新读取材料、调用工具或补做分析：

   ```text
   只回顾刚才已经完成的执行过程，不要再读取文件、调用工具或补做分析。
   请列出你实际读取或使用过的所有 skill、触发它们的任务信号，以及它们具体改变的步骤、判断或输出。
   没有使用则明确说明；只报告实际发生过的行为，不要事后补选 skill。
   ```

3. 回访必须区分作为任务对象被读取的 skill、仅被考虑或加载的 skill，以及确实改变执行行为的 skill。目标名称出现在任务中、读取过入口或能够复述规则，都不能单独证明实际采用。
4. 需要运行多个案例时分别完成首次执行与回访；不要先向所有实例公开测试目标，再收集所谓盲态结果。

## 判断证据

三类证据支持不同判断，不能相互替代；事后自述只用于解释前两类更直接的证据：

1. **执行轨迹**证明实际读取和动作，包括入口和 references、脚本或工具、停下询问和交接；仅有读取不能证明规则被采用。
2. **可观察产物**证明最终表现出的流程、判断、权限边界和完成证据；仅有相似产物不能证明变化由哪个 skill 导致。
3. **事后自述**解释实例认为自己为何选择 skill、哪些行为受其影响；它可能遗漏事实或产生事后合理化，不能单独证明读取、采用或改善。

对每个案例分别记录预期触发边界、初始条件、实际选择的 skills、读取与采用证据、产物表现、偏差、阻塞和干扰因素。结论只使用“已观察支持”“未观察到”或“证据不足”；不要仅凭自述、一次成功输出或计划中的检查宣布验证通过。

探测回答的是不同层次的问题：

- **发现**：agent 是否自行选择并加载目标 skill。
- **采用**：skill 是否具体改变了步骤、判断、资源路由或权限处理。
- **结果**：产物是否满足当前设计契约。
- **改善**：变化是否由目标 skill 带来并优于 baseline；通常需要受控对照和独立审阅。

## 进入设计契约

1. 将稳定的运行观察放入“现实案例或观察行为”，把环境差异、样本不足和无法解释的偏差保留为未知项或残余风险。
2. 由观察结果修订触发、Workflow Map、判断、权限、资源归位或验证义务；不把临时任务、完整轨迹和回访问答自动复制进长期 skill 内容。
3. 如果探测暴露的是目标 skill 的实现或分发问题，将证据交给当前环境的维护入口；本 skill 仍只负责形成可实现设计。
4. 当前环境没有独立执行能力时，交付可复现的任务、隔离条件、预期行为差异和证据要求，并明确标记为未运行验证。
