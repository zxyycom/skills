# 设计哲学

本文件保存 Skill 的深层设计理由。只有短操作文件不够时再读。

## Skill 是行为系统

Skill 捕获的是可复用的行为模式，不是一堆说明。用户通常有隐性的流程：看什么、何时判断、检查什么、何时询问、什么算完成。创建 Skill 就是把这个流程提取出来，变成另一个 agent 能稳定运行的系统。

## Context Is Everything

上下文质量和上下文长度同样重要。坏 Skill 会让 agent 吞下低信号材料、重复发现工具，并把重要状态留在聊天历史中。好 Skill 会把当前步骤需要的 reference、tool、asset、contract 和人类输入路由过来。

## 文件压缩状态

强流程不能依赖模型记住长对话。中间状态应该落到文件和 contract 中。下游步骤应该能通过检查前置产物知道发生过什么。

## References 和 Scripts 都是工具

References 是认知工具。Scripts 是执行工具。Assets 是生产工具。Contracts 是交接工具。Skill 设计的核心，是决定每一步拿到什么工具。

## 人机介入

人机介入不是自动化失败。需要方向、判断、批准或知识时，它就是 workflow 的一部分。Skill 必须设计用户看到什么，以及反馈如何影响下游。

## Memory 不是日志

Memory 是每个目标 Skill 的持续迭代机制，不是本规划流程的收尾日志。只写会改变未来运行的内容。反复成立的经验升级进 Skill；过期沉积要删除。

## 根本德性

根本德性是可预测的过程。好 Skill 不保证每次输出相同，但会让 agent 用同一条稳定路径完成工作。
