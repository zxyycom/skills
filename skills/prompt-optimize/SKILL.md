# Prompt Optimize

## 目标

把 prompt、规则、任务、需求、模板、工作流和 agent 指令等结构化文本, 改成目标清晰、范围明确、正向可执行、结构显式、边界和验收标准清楚、阅读脉络符合用途、便于复用维护的文本。

## 文件分工

1. 本文件: 入口、读取策略、最小执行协议和完成检查。
2. [workflows.md](references/workflows.md): 生成、改写、审阅、直接编辑和交付流程。
3. [rewrite-rules.md](references/rewrite-rules.md): 按改写管线组织用途分流、骨架选择、行为审计、内容归位、规则收敛、表达落地和沉淀判断。
4. [principles.md](references/principles.md): 规则背后的原理、取舍和长期维护理由, 按“原理或思考 -> 行为”说明。
5. [agent-tasks.md](references/agent-tasks.md): worker、explorer、并行 agent 和子 agent 任务结构。

## 读取策略

先读本文件。按当前任务只加载必要引用:

1. 改写、审阅或生成 prompt、规则、需求、模板时, 读取 [workflows.md](references/workflows.md)。
2. 需要执行具体改写判断时, 读取 [rewrite-rules.md](references/rewrite-rules.md), 按其中的改写管线推进。
3. 需要写 worker、explorer、并行 agent 或子 agent 任务时, 读取 [agent-tasks.md](references/agent-tasks.md)。
4. 用户要求解释原理、争议取舍、内容所有权或长期维护理由时, 读取 [principles.md](references/principles.md)。
5. 需要整体审阅 skill 或判断引用一致性时, 读取入口文件和所有直接引用文件。

## 最小执行协议

1. 定位目标:
   - 用户指定文件或目录时, 只处理指定范围。
   - 用户只给出项目或工作区时, 查找 `AGENTS.md`、`CLAUDE.md`、`SKILL.md`、规则文件、prompt 模板和 agent 任务模板。
   - 多个高风险目标优先级不清时, 用一个简短问题确认范围。
2. 按改写管线处理:
   - 先定任务出口: 用途、使用者、目标状态和风险等级。
   - 再选文档骨架: 执行、流程、规则、说明、参考、契约、检查、评估或协作。
   - 做行为审计, 给内容找 owner, 收敛规则关系。
   - 处理负向描述, 落地表达和格式, 最后判断是否值得沉淀。
3. 交付结果:
   - 直接编辑目标文件时, 保持改动范围最小。
   - 只做审阅时, 使用“问题 -> 建议调整 -> 理由”。
   - 无法可靠推断替代方案时, 说明缺口并问一个具体问题。

## 冲突处理

默认以用户当前目标和显式偏好为主, 但按风险处理冲突:

1. 涉及安全、权限、合规、事实真实性、不可逆操作、更高优先级指令或明确契约时, 先说明冲突并确认可行路径。
2. 与项目约定、已安装依赖或相邻模块做法冲突但风险可控时, 可以按用户目标处理, 并在交付中汇报取舍。
3. 无法判断冲突风险时, 问一个具体问题确认。

选择具体方案时参考: 用户目标、项目约定、已安装依赖、相邻模块做法和行业常见方案。

## 完成检查

1. 是否按任务出口、文档骨架、行为审计、内容 owner、规则关系、负向描述、表达格式和沉淀判断的顺序处理。
2. 当前信息和行为引导是否符合文档实际用途。
3. 核心判断是否只有 owner 位置完整解释。
4. 安全、权限、合规、真实性和不可逆操作边界是否保留, 且有正确做法时配套正向路径。
5. 重复、冲突、模糊表达和复杂格式是否已收敛。
6. 坏示例、旧版描述、临时提醒和单次问题是否只作为诊断材料使用。
