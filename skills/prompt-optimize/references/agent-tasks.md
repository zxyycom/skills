# 子 Agent 任务优化

用于为 worker、explorer、并行 agent 或子 agent 写任务。目标是压缩上下文、明确所有权, 并让输出可合并。

## 任务模板

```text
目标: <正向描述要完成的任务>
范围: <负责的文件、模块或问题边界>
方法偏好: <项目已有约定、推荐工具或实现路径>
协作边界: <并行工作时的所有权和已有改动保护>
输出: <发现、改动文件、验证结果、阻塞点; 优先用 JSON 或有序列表>
```

## 编写步骤

1. 给每个 agent 一个清晰的文件、模块或决策所有权。
2. 拆分并行任务时, 让范围互斥或写清合并顺序。
3. 给出必要上下文, 不泄露预设答案、隐藏结论或期望修复。
4. 把坏示例提炼成目标状态、推荐路径或验收标准。
5. 要求输出包含发现、改动、验证和阻塞点。

## 检查标准

1. 每个 agent 是否只有一个主要目标。
2. 并行 agent 的职责是否没有重叠。
3. 输出是否能被主 agent 直接合并或决策。
4. 任务是否避免携带错误清单和旧版本描述。
5. 边界是否写成目标状态、推荐路径和验收标准。

## 输出 JSON 示例

```json
{
  "findings": ["<具体发现或空数组>"],
  "changes": ["<改动文件或空数组>"],
  "verification": ["<已运行检查或空数组>"],
  "blocked": ["<阻塞点或空数组>"]
}
```
