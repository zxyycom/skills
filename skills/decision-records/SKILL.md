# Decision Records

## 目标

用公开、可版本化的 Markdown 保存已经确认并成为后续工作依据的决策，使人类和 agent 能快速找到当前已生效判断，并在需要时回放形成原因。

决策记录承接原因、取舍和演进关系；代码、配置、规范和项目文档继续承接当前事实与行为。记录过程只在重要长期取舍形成时介入，不反复打断原任务。

## Owner 与读取

1. 本文件承接发现、确认、读取、写入和交付流程。
2. [decision-record-rules.md](references/decision-record-rules.md) 是唯一固定契约，承接全部目录、字段、状态、关系、索引和维护事务。
3. `scripts/decision-records.mjs` 承接确定性的结构检查、状态列表和活动索引同步。
4. 目标工作区的 `docs/decisions/` 只保存索引和实际决策；项目专属的更高门槛写入 `AGENTS.md` 或相关行为 owner。

按任务逐步读取：

1. 先读目标工作区的指令文件和相关行为 owner。
2. 恢复上下文或评估候选决策时，只读索引和相关 `active` 记录。
3. 创建、修订、改变状态、显式修复或结构审阅前，完整读取固定契约。
4. 只有审计演进原因时才沿直接关系读取历史；归并决策是默认追溯边界。

## 执行流程

### 1. 区分任务意图

1. 恢复、解释、检查或审阅保持只读。
2. 用户明确要求记录或修订一项内容完整、长期适用的决策时，将本次要求作为写入确认。
3. 用户要求修复非标准格式时，只将已经说明的修复范围视为授权。
4. 普通工作中只在自然更新或交付阶段提出重要候选；必须先确定该取舍才能继续时再即时提出。

写入内容、长期范围或确认意图仍不清楚时，先给出概览并只问一个确认问题。

### 2. 恢复当前判断

1. 从活动索引按影响面定位相关 `active` 记录，不按日期遍历全部文件。
2. 将 `active` 作为已经确认并成为后续工作依据的已生效决策，不把它解释为进行中的任务。
3. 对照当前行为 owner，区分“现在怎样做”和“为什么这样做”。
4. 当前任务指令与既有判断不同时，先按当前指令完成本次任务；只有确认它将长期改变后续工作时才更新决策历史。
5. 多条 `active` 记录互相冲突，或与行为 owner 冲突时，报告一致性问题；没有确认依据时不自行选择。

### 3. 提出候选决策

候选判断需要同时具备长期影响和回放价值。依次确认：

1. 缺少记录时，未来是否难以解释为何采用当前方案而不是其他合理方案？
2. 它是否会改变后续行为、owner、边界、兼容性、风险处理或验收方式？
3. 它是否包含可复用取舍，而不是事实、状态、任务或执行结果？

候选尚未确认时，给出一份短概览：

1. 当前决定。
2. 核心理由。
3. 适用与重评条件。
4. 影响范围。
5. 与既有决策的关系。

让用户确认、修改或放弃。任务局部选择、一次性例外、建议和开放问题继续留在原 owner。

### 4. 写入或修复

1. 完整读取固定契约，并使用用户指定的决策目录；未指定时使用 `<workspace>/docs/decisions/`。
2. 已确认写入且目录不存在时，按契约创建最小体系。
3. 目录不符合契约时，先报告差异、可保留语义、修复范围和不确定项；得到显式修复确认前保持只读。
4. 得到确认后，按契约完成新增、状态变化、直接关系、归并或格式修复，并同步当前行为 owner 与活动索引。
5. 修复中的不确定项会改变决策含义、当前有效性、关系或 owner 时，请用户判断。

### 5. 验证与交付

1. 按固定契约完成语义检查和结构检查，确认没有未解释的有效决策冲突或不必要的敏感信息。
2. 同步活动索引并运行 `check`，修复到退出码为 `0`。
3. 等待确认时，交付候选概览并明确尚未写入。
4. 已写入时，列出新增、修订、归并、迁移或状态变化，说明沿用的已生效判断和同步的行为 owner。
5. 没有写入时，说明是门槛不足、尚未确认、只读要求，还是仍在等待修复授权。

## CLI

从 skill 目录运行，或使用脚本的绝对路径：

```text
node scripts/decision-records.mjs check --root <workspace-root>
node scripts/decision-records.mjs list --root <workspace-root>
node scripts/decision-records.mjs list --all --root <workspace-root>
node scripts/decision-records.mjs list --status amended,superseded --root <workspace-root>
node scripts/decision-records.mjs sync-index --write --root <workspace-root>
```

不写命令时默认执行只读 `check`。使用 `--decisions-dir <path>` 指定非默认目录；`sync-index` 只有带 `--write` 时更新 `## 活动决策`。

## 完成标准

1. 只有明确确认的长期取舍进入决策记录。
2. 活动索引能够按影响面定位全部已生效的 `active` 记录。
3. 当前判断、历史关系、行为 owner 和索引保持一致。
4. 非标准格式在未获授权时只形成修复报告，获授权后收敛到唯一固定契约。
5. CLI 检查通过，交付包含实际语义检查和校验结果。
