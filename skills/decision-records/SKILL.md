# Decision Records

## 目标

用显式、可版本化的 Markdown 保存长期决策的目的、形成背景、采用方向和必要关系，用 JSON 索引保存当前仍作为后续工作依据的决策及其精简摘要。让人类和 agent 在形成可能产生长期影响的决定前快速恢复希望达成的结果、当时约束和当前方向，在需要时低成本回放理由，并避免后续维护静默偏离。

决策记录不承接通用记忆、任务日志或当前实现状态；代码、配置、规范和项目文档继续承接当前事实与行为。本文中的行为 owner 指权威承接这些事实或规则的文件。skill 只在重要长期取舍形成时介入，不反复打断原任务。

## 内容 owner

1. 本文件承接发现、语义恢复、偏离判断、候选确认、CLI 选择和按任务出口交付。
2. [decision-record-rules.md](references/decision-record-rules.md) 是唯一固定契约，承接决策根目录、索引 schema、稳定身份、正文格式、关系、生命周期、维护事务和 CLI 精确语义。
3. `scripts/decision-records.mjs` 是查询、关系追溯、当前成员变更、派生字段同步和结构检查的确定性接口。
4. 目标项目的决策根目录承接当前索引和 Markdown；项目专属的更高记录门槛写入 `AGENTS.md` 或相关行为 owner。

## 主动读取

1. 先读目标工作区的指令文件和相关行为 owner。
2. 在形成可能产生长期影响的决定或处理与既有决定的冲突前，确定决策根目录并定位当前索引。根目录整体尚不存在时，将集合解释为尚未初始化；已有决策材料但索引缺失或无法解析时，说明当前成员无法可靠恢复。
3. 先用 `list` 获取当前成员、生命周期和结构诊断，再从 JSON 当前索引的路径、标题、目的、背景和决策摘要定位相关判断，不按日期扫描全部 Markdown。
4. 摘要足以判断时停止；需要完整理由时读取命中的 Markdown，需要审计演进时使用 `trace`。只有审计演进原因时才沿关系读取历史，归并记录是默认追溯边界。
5. 恢复当前判断、归类偏离和提出未确认候选时不读取固定契约；创建、修订、逻辑归档、归并、显式修复或结构审阅前完整读取固定契约。

## 执行流程

### 1. 确定任务出口

1. 恢复、解释、检查或审阅已有决策时，交付查询或审阅结果；当前指令没有要求维护时，不扩大改动范围。
2. 当前指令已经明确长期判断、维护对象和影响范围时，将其视为已经确认并直接维护，不重复询问；新增、调整或退出当前集合前，先告知将写入或改变的判断及集合变化。
3. 普通工作中形成新的重要取舍时，先恢复相关决策，在自然更新或交付阶段提出候选；只有必须先确定该取舍才能继续时才即时提出。
4. 修复非标准格式时，先清点并报告拟转换范围；当前指令只授权已经说明并确认的范围。

写入内容、长期含义、维护对象或影响范围仍不清楚时，先给出最小概览并只问一个确认问题。

### 2. 恢复当前判断

1. 从当前索引的 `path`、`title`、`purpose`、`background` 和 `decision` 定位并简要恢复相关当前记录。
2. 将索引成员解释为已经确认并成为后续工作依据的决策；不在索引中的记录只用于历史回放。
3. 从 `目的` 恢复希望达成或维护的长期结果，从 `背景` 理解当时为何需要作出选择，从 `决策` 恢复采用的方向，再对照当前事实和行为 owner 判断当前任务是否仍应沿用；当前成员是后续工作的默认依据，不是脱离目的和背景的永久规则。
4. 将当前任务与相关决策归为以下一种情况：
   - 对齐：按当前判断继续。
   - 明确的一次性例外：按当前指令完成本次任务，说明例外范围，并明确当前索引和历史未改变。
   - 长期修订：形成候选概览；当前指令已经明确要求长期记录或修订时复用该确认，否则等待确认后再写入。
   - 一致性问题：当前记录互相冲突、记录与行为 owner 冲突，或偏离性质不清且会影响实质改动时，报告冲突并请求判断。
5. 当前指令只改变本次任务的执行方式时，归为一次性例外；只有用户已经决定改变后续工作的默认依据时，才归为长期修订。
6. 当前事实与记录背景已经明显不同时，说明变化、受影响判断和需要确认的新取舍，不机械沿用旧背景。
7. 没有当前指令或既有记录提供确认依据时，不自行选择冲突方向，也不静默改变长期行为。
8. `list` 或 `trace` 成功只表示返回了当前可恢复结果。warning 可能影响相关主题时，不根据缺失记录、关系或结果断言它们不存在；说明不完整性，并按需检查相关 Markdown、索引或运行严格 `check`。

### 3. 提出候选决策

候选判断需要同时具备长期影响和回放价值。依次确认：

1. 它是否会持续作为后续工作依据，或改变长期行为、owner、边界、兼容性、风险处理或验收方式？
2. 缺少记录时，未来是否难以恢复为何采用当前方案？
3. 它是否包含可复用判断或取舍，而不是事实、状态、任务或执行结果？
4. 它是否是一项能够独立修订或替代的判断？需要分别演进的判断分别处理。

当前指令已经明确表达这项长期判断时，复用该确认并进入维护。否则只给出形成判断所需的最小概览：

1. 目的。
2. 背景。
3. 采用的决策。
4. 与既有决策的直接关系；没有关系时省略。

让用户确认、修改或放弃。任务局部选择、一次性例外、建议和开放问题继续留在原 owner。

### 4. 写入或修复

1. 完整读取固定契约，并据此确定决策目录和维护事务。
2. 按固定契约区分编辑性修正与决策演进，并选择新增、修订、替代、判定无效、归并或独立逻辑归档事务。
3. 准备 Markdown、真实关系和需要同步的行为 owner，再通过 CLI 维护当前成员和 JSON 派生字段，不直接维护生成字段。
4. 首条已确认记录同样通过 `activate` 进入当前集合，不增加独立初始化步骤；精确初始化条件和失败恢复以固定契约为准。
5. 目录不符合契约时，先报告差异、可保留语义、修复范围和不确定项；得到修复范围确认后再执行转换。
6. 修复中的不确定项会改变决策含义、当前成员、关系或 owner 时，请用户判断。

### 5. 验证与交付

1. 确认记录表达了已经确认的目的、背景和采用方向，并检查当前事实、行为 owner、相关决策和本次集合变化之间没有未解释的冲突或不必要的敏感信息。
2. 完成写入或迁移后同步 JSON 派生字段并运行严格 `check`，修复到退出码为 `0`。
3. 等待确认时，交付候选概览、未确认项和“尚未写入”的状态。
4. 已写入时，列出新增、修订、替代、判定无效、归并、逻辑归档或迁移，说明当前集合变化、偏离分类、行为 owner 检查或同步和校验结果。
5. 只恢复或审阅时，交付相关当前判断、偏离分类、查询 warning 及其对结论完整性的影响。

## CLI

从 skill 目录运行，或使用脚本的绝对路径：

```text
node scripts/decision-records.mjs list [--archived|--all] --root <workspace-root>
node scripts/decision-records.mjs trace <topic/file.md> [--direction <direction>] [--depth <n>] --root <workspace-root>
node scripts/decision-records.mjs check --root <workspace-root>
node scripts/decision-records.mjs sync-index --write --root <workspace-root>
node scripts/decision-records.mjs activate <topic/file.md> --root <workspace-root>
node scripts/decision-records.mjs archive <old.md...> [--by <new.md>] --root <workspace-root>
```

用 `list` 和 `trace` 恢复决策，用 `check` 严格检查结构；读取固定契约后再使用写命令。`--decisions-dir <path>` 可以选择工作区内的非默认决策根目录。命令参数、合法值、初始化、warning、成员变更和失败恢复的精确语义以固定契约与 `--help` 为准。

## 完成标准

### 恢复或审阅

1. 已从当前索引和必要的 Markdown 恢复相关判断，并结合目的、背景、当前事实和行为 owner 判断是否沿用。
2. 当前任务的偏离已经归类；查询 warning 已被披露，且结论没有超出可恢复结果能够证明的范围。
3. 交付与当前查询或审阅目标一致，没有扩大为未要求的维护。

### 候选决策

1. 候选同时满足长期影响和回放价值，粒度能够独立演进。
2. 确认状态清楚；未确认候选没有进入当前索引。

### 维护或迁移

1. 只有明确确认的长期取舍进入当前索引；集合变化前已经告知用户，非标准格式转换没有超出确认范围。
2. 精确维护或结构审阅前已经读取固定契约，历史语义通过正确事务和真实关系保留。
3. 当前集合、决策内容、行为 owner 和按需关系没有未解释的冲突。
4. 严格 `check` 返回 `0`，交付说明实际语义检查、集合变化和校验结果。
