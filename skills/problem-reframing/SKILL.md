# Problem Reframing

## 目标

在继续求解前, 判断当前问题是否值得并适合按现有方式解决, 再把有证据支持的问题框架交回当前任务。

## 判断对象

问题框架描述当前工作怎样连接到真实结果, 至少包括:

1. 希望改变的结果、真实消费者和发生场景。
2. 已知事实、关键约束和仍未验证的前提。
3. 当前问题的对象、边界、因果位置与抽象层级。
4. 判断问题已经解决的证据。
5. 当前解法及其隐含的问题表述。

把当前解法视为候选路径, 不让它反向充当目标。跳出局部也不等于无限扩大范围; 最终选择应是能够解释真实结果的最小问题框架。

## 执行流程

### 1. 判断是否需要重构

1. 明确当前任务出口与用户授权: 分析、决策、设计、实现、修订、审查或其他交付。
2. 用户明确要求重新审视问题, 或当前证据出现以下框架风险时继续:
   - 局部规则、例外、补偿机制或协调成本持续增加。
   - 多次优化当前解法, 但真实消费者或可观察结果没有相应改善。
   - 需求或成功指标描述的是解法形式, 没有说明真实结果。
   - 当前层级只能处理症状, 或多个局部目标彼此冲突。
   - 关键前提长期按事实使用, 却没有证据支持。
3. 目标、前提、边界和责任已经清楚, 且问题确实局部时, 结束重构并直接回到当前任务, 不为简单工作制造额外分析。

### 2. 恢复当前框架

1. 用一句话写出当前正在回答的问题, 区分“想得到什么结果”和“准备怎样实现”。
2. 找到真实消费者、发生场景、期望变化和成功证据; 没有外部消费者时, 明确内部决策者或后续流程。
3. 将上下文分为事实、明确约束、基于证据的推断、未验证前提和已经选中的解法。
4. 标出当前问题边界、所在抽象层以及它能够直接影响的变量。

只恢复足以改变当前选择的上下文。

### 3. 形成替代框架

沿可能改变结论的方向检查当前框架:

1. **结果**: 即使当前解法成功, 是否会产生消费者真正需要的变化?
2. **必要性**: 当前子问题是否必须解决, 还是可以通过删除需求、接受约束或改变流程消除?
3. **前提**: 哪些前提决定了问题存在? 它们是事实、历史选择, 还是尚未验证的假设?
4. **边界与因果**: 当前范围是否只能补症状, 或把能够独立处理的问题绑在一起? 哪个位置能够控制原因?
5. **抽象层**: 上移到契约、模型、owner、策略或流程后, 能否把复杂功能变成简单应用?
6. **替代性**: 能否通过测量、试验、人工流程、现有能力或不采取动作获得同一结果?

根据检验结果只形成有现实依据的候选:

1. **保留**: 原问题成立, 继续处理当前解法。
2. **缩小**: 只保留能直接产生目标结果的部分。
3. **改写**: 用真实结果替换由既有解法定义的问题。
4. **上移**: 把问题交给能够控制根因的更高抽象层、责任 owner 或流程。
5. **放弃**: 证据表明问题不值得解决、已经由其他机制覆盖, 或成本明显高于结果价值。

没有证据表明某个检查方向会改变选择时停止展开。每个实际候选写清目标结果、改变的前提或边界, 以及支持或反驳它所需的证据。

### 4. 选择问题框架

1. 比较候选与真实结果的直接程度、解释力、能够消除的复杂度、实施与协调成本、风险、可逆性和证据强度。
2. 优先选择能够消除根因且范围最小的框架, 不因“更宏观”本身偏好大规模重构。
3. 证据不足时, 选择能够区分候选的最小调查、实验或度量, 暂停优化尚未成立的解法。
4. 证据足以区分候选时, 明确选择保留、缩小、改写、上移或放弃, 并说明真正改变结论的依据。

### 5. 回到当前任务

1. 交付选定方向、重新表述的问题、关键依据、约束与未知项, 以及下一步动作。
2. 保留原框架时直接继续原任务; 新框架仍在当前授权内时, 将它作为后续分析、设计或实现的输入。
3. 新框架会实质改变用户已经明确的目标、范围或外部状态时, 先交付重构结论并请求所需判断; 问题重构本身不扩大修改授权。
4. 输出服从当前任务的行为 owner 和格式, 不默认生成独立的长篇框架报告。

## 边界

1. 以用户明确请求或可观察的框架风险作为触发依据, 不因存在另一种解法就自动重构问题。
2. 以上下文支持的真实结果作为上层目标, 不用个人偏好、宏大叙事或假想需求取代它。
3. 将调查扩展到能够解释结果的最小边界, 对框架清楚的直接任务保持局部和简短。
4. 后续领域分析、设计、实现和验证继续由当前任务的行为 owner 承接。
5. 保留用户已经明确的目标、约束和授权; 关键选择无法可靠判断时, 只请求或调查会改变结论的信息。

## 完成检查

1. 是否有明确请求或框架风险支持本次重构, 否则已及时回到局部任务?
2. 是否恢复了真实结果、事实、约束、前提、边界和当前解法?
3. 候选是否只覆盖会改变选择的方向, 并由现实证据支持?
4. 是否明确选择保留、缩小、改写、上移或放弃, 并给出改变结论的依据?
5. 选定框架能否直接成为下一步输入, 且没有越过用户授权或替代当前行为 owner?
