# 将文档优化到 AI-ready 状态的原则

本文件解释怎样让 AI 从实际获得的文档文本中准确恢复含义并可靠使用, 以及为什么这些调整通常也会改善人的阅读和维护。具体执行路径由入口文件承接。每节按“原理或思考 -> 行为”组织。

## 单一内容源原则 -> AI-ready 不建立第二份真相

原理: AI 通常读取由人维护的现有文档。另建内容不同的“AI 版”会产生第二事实源, 让更新、引用和判断逐渐分叉。AI-ready 的目标是在权威内容中形成可恢复的语义结构, 而不是复制一份机器专用真相。

行为:

1. 先识别 AI 完成预期任务必须获得的信息, 再判断哪些解释、索引或结构只需按需展开。
2. 事实、规则和关键判断只在稳定 owner 位置完整表达一次。
3. 主文档承接权威主线; 摘要、附录、参考和结构化块通过引用或明确映射连接主线。
4. 外部契约确实要求独立机器产物时, 将其视为可追溯的消费视图, 不让它成为内容不同的第二事实源。

## AI 实际消费原则 -> 先建立 AI 消费契约

原理: AI 只能依据实际获得的文本和可解析结构恢复含义。文档是否 AI-ready 取决于目标 AI 拿到什么、需要完成什么, 不能由“结构看起来清楚”或“理论上可以访问”推断。

行为:

1. 先确定文档用途、目标 AI、实际可用文本、预期操作和可观察结果。
2. 显式表达会改变 AI 理解或行为的对象、关系、范围、权威性、优先级、依赖、条件和例外。
3. 必需信息不在 AI 实际可用文本中时, 在当前文档补足最小必要语境或把交付链路作为外部依赖暴露。
4. 不把上下文组装、索引或召回问题伪装成文档已经解决的问题。

## AI 信息恢复原则 -> 显式表达关键关系

原理: AI 对视觉位置、默认常识和隐含语境的利用不稳定。表格列对齐、目录树缩进、颜色和普通思维导图的空间距离可能帮助人类浏览, 却不能替代文本中的对象、关系、方向和范围。

行为:

1. 明确术语指代、适用范围、优先级、依赖、条件、例外、因果方向以及当前与历史状态; 区分权威内容、引用、示例和外部材料。
2. 标题、列表、表格和图示用于组织已经表达清楚的语义, 不单独承担关键含义。
3. 章节、列表项和表格行在脱离相邻视觉位置后仍要能识别主题、对象和适用范围。
4. 简单对照使用每行可独立理解的表格; 复杂结构化输入输出和机器契约遵循既有格式, 没有契约时再选择 JSON、扁平路径或 nodes/edges 等稳定表达。

## 解空间锚定原理 -> 先建立目标框架, 再保留必要边界

原理: LLM 即使准确理解“不使用 X”, X 及其常见用法、修正路径和反例仍已进入当前上下文。生成替代方案时, 模型可能继续围绕 X 的正确做法或反面搜索, 把解空间限制在由 X 建立的局部框架; 更优方案有时需要从目标、约束和结果重新建立解题路径, 而不是继续解释 X。

行为:

1. 先从目标状态、必要约束和预期结果建立正向框架, 再处理“不要 X”。
2. 判断 X 是迁移残留、低频极端方案、默认倾向冲突还是硬性边界。
3. 当前推荐路径明确时, 写“使用 Y 完成目标”, 不围绕 X 继续展开替代方案。
4. X 是默认容易回落的方案、明确契约或重要边界时, 保留一次必要边界并补正向做法。
5. 推荐路径、接口契约或合法值集合已经自然排除 X 时, 不再把 X 写成额外特例。

## 文档用途原则 -> 阅读结构服务真实用途

原理: 说明、规则、流程、参考和契约需要不同的阅读顺序与信息密度。AI 能否正确恢复和使用内容, 取决于结构是否服务真实用途, 而不是是否套用了固定模板。

行为:

1. 按用途选择骨架:

   | 用途 | 主骨架 |
   | --- | --- |
   | 执行 | 目标、范围、输入、步骤、边界、验收、输出 |
   | 流程 | 前置条件、状态转移、步骤、失败处理、恢复路径、验收 |
   | 规则 | 触发条件、规则、例外、优先级、检查方式 |
   | 说明 | 问题引入、概念、推理递进、取舍、结论 |
   | 参考 | 索引、概念、条目、示例、相关链接 |
   | 契约 | 字段、合法值、约束、兼容性、校验方式 |
   | 检查或评估 | 检查项、通过标准、失败表达、残余风险 |
   | 协作 | 目标、范围、所有权、协作边界、输出字段、阻塞表达 |

2. 文本同时承担多种用途时, 以最容易造成误用的用途为主骨架, 将其他用途放入清楚的章节、附录或参考。
3. 只保留会改变理解或使用的骨架成分, 不为套模板补无关字段。
4. 阅读者与消费方式判断只服务于内容取舍、结构和表达; 不影响理解时不把分析过程写进正文。

## 文档主承诺原则 -> 正文重心要兑现主承诺

原理: 标题、副标题、序言、开头段、一级标题、触发条件、前置条件、资源名、字段集合、适用范围和输入输出描述会先于细节建立预期。后续篇幅、层级、结论、验收或校验方式如果集中到另一个主题, AI 会围绕错误中心恢复含义, 人类维护者也会沿错误脉络阅读。owner 判断只能解决内容完整解释的位置, 不能单独判断整篇文档的重心是否偏移。

行为:

1. 从标题、副标题、序言、开头段、一级标题、触发条件、前置条件、资源名、字段集合、适用范围或输入输出描述推断文档主承诺。
2. 对照正文反复解释、篇幅最多、层级最高、结论落点、验收标准和校验方式, 检查实际重心。
3. 正文偏离主承诺时, 移动、降级或删减偏离内容。
4. 标题、开头、适用范围或契约入口过宽时, 收窄入口承诺。
5. 偏离内容有独立维护价值时, 移到对应 owner; 当前文档只留摘要、触发条件或引用。

## 内容所有权原则 -> 一个判断只在一个 owner 位置完整解释

原理: 同一规则在多个位置完整展开, 会带来重复、漂移和局部修补。

行为:

1. 按使用者的查找路径、触发上下文、验收责任和后续变更位置确定 owner。
2. 事实、规则和关键判断由对应 owner 完整承接; 解释材料、消费视图和相邻文档不复制成新的事实源。
3. 非 owner 位置只保留摘要、触发条件或引用。

## 语义保真原则 -> 优化表达不替代领域判断

原理: 优化文档会重新组织或显式化已有内容, 但不会因此获得补造事实、替换要求或代替领域 owner 作出取舍的权限。把未经确认的推断写成确定结论, 会让 AI 更稳定地依赖错误内容, 也会误导后续人类维护者。

行为:

1. 区分结构与表达优化、事实补充、要求变化和领域决策。
2. 原意、事实或取舍不确定且不同解释会改变结果时, 暴露歧义和缺失依据, 不自行补造。
3. 确需改变含义时显式标出变化, 并由相应事实或决策 owner 确认。

## 人类可维护性约束 -> AI 优化后仍保持自然阅读

原理: 清楚目标、稳定术语、显式关系和减少歧义通常也会帮助人类。人类可读性不是第二套对称优化目标, 但 AI 优化不能把原本可维护的文档压成机械字段堆叠, 也不应删除维护者理解取舍所需的脉络。

行为:

1. 先兑现 AI-ready 主承诺, 再检查人的阅读和维护是否明显退化。
2. 说明型内容保留必要的问题引入、概念递进、取舍和结论; 结构化文档的标题、字段和步骤仍让人能直接理解。
3. 细节很多时使用渐进展开和导航; 复杂关系确实更易通过图示理解时使用显式关系图或 flowchart, 但关键语义仍存在于文本或图源。
4. 纯粹面向人的措辞润色、视觉美化或体验设计不进入本 skill 的主流程。

## 可验证性原则 -> 用 AI 的实际结果证明 ready

原理: “更好”“高级”“合理”“适当”会把判断留给使用者临场猜测, 让输出难以复现和验收。AI-ready 不能只由结构整齐或字段齐全推断, 还要回到文档真实用途检查 AI 能否恢复并使用内容。

行为:

1. “更好”改成目标状态和验收标准。
2. “高级”改成受众、复杂度、示例深度和输出格式。
3. “合理”改成决策优先级; “适当”改成触发条件、范围和上限。
4. 重大或高风险改写选择代表性 AI 阅读或使用任务, 检查它能否从实际文本得到预期结果。
5. AI 验证通过后再检查文档是否仍便于人类阅读和维护, 不把人类侧检查升级成对称主目标。
6. 目标已有格式、链接、schema 或其他文档校验时运行相关检查; 机械通过不能替代 AI 的语义与使用结果检查。

## 长期沉淀原则 -> 错误样本先当诊断材料

原理: 坏示例、旧版描述、迁移残留、临时提醒和单次问题通常说明当前规则哪里没覆盖, 但它们本身不是长期规则。

行为:

1. 能提炼成当前目标、推荐做法、边界或验收标准时, 改写后保留。
2. 只是已有规则的重复例子时, 合并到已有规则或删除。
3. 只描述一次性错误、历史过程或旧版本做法时, 不写入长期文档。

## 风险分级原则 -> 冲突先按后果处理

原理: 用户目标、项目约定、工具默认路径和安全边界可能互相冲突。不同冲突的后果不同, 不能用同一种策略处理。

行为:

1. 涉及安全、隐私、机密性、权限、合规、事实真实性、不可逆操作、更高优先级指令或明确契约时, 先说明冲突并确认可行路径。
2. 项目约定、依赖或相邻模块做法冲突但风险可控时, 可以按用户目标处理, 并在交付中汇报取舍。
3. 无法判断冲突风险时, 问一个具体问题确认。
