# 调查报告固定契约

本文件只定义 `investigation-report` 的落盘结构和检查器语义。何时调查、怎样取得证据、哪些场景内容必须补足、何时追加或拆分以及如何判断内容质量，由 [SKILL.md](../SKILL.md) 承接。

## Owner 与目录

1. 主题文件顶部的 `调查信息` 只保存核心问题、状态和最新报告时间。
2. `调查报告` 中每个三级标题是一份形成于特定时点的完整报告；最新报告承担当前认识，旧报告保留形成时快照。
3. `investigation-index.md` 只投影主题文件的标题、核心问题、状态和最新报告时间，用于发现，不保存结果摘要。
4. `scripts/check-investigations.mjs` 只读检查路径、固定结构、时间和索引投影，不写入文件，也不判断内容质量。

默认目录：

```text
docs/investigations/
├── investigation-index.md
└── <topic-id>/
    └── <semantic-slug>.md
```

`topic-id` 和 `semantic-slug` 使用英文小写、数字与连字符组成的 kebab-case。一个文件只承接一个稳定核心问题，路径创建后不因标题、状态或追加报告改变。除索引外的 Markdown 文件都必须位于一层主题目录中。首次创建主题时同时建立调查根目录和索引。可以用 `--investigations-dir` 选择工作区内的其他调查根目录，但同一集合始终使用同一根目录。

## 索引

索引首个非空行固定为 `# 调查索引`。每个主题使用一个与目录名相同的二级标题，每个主题文件使用以下条目：

```markdown
## codex

- [项目 Shell MCP 注册调查](codex/project-shell-mcp-registration.md)
  - 核心问题: 为什么项目 Shell MCP 没有注册到可用工具列表？
  - 状态: 调查中
  - 最新报告时间: 2026-07-21T14:05:30+08:00
```

规则：

1. 链接使用相对调查根目录的 POSIX 路径，链接文字逐字等于主题文件 H1 的纯文本。
2. 三个字段按示例顺序各出现一次，并逐字投影主题文件的 `调查信息`。
3. 每个主题文件有且只有一个索引条目，每个条目存在对应文件。
4. H1 与首个主题之间可以有简短说明；主题内部不增加额外层级或打断条目的内容。

## 主题文件

主题文件首个非空行是唯一 H1，前两个 H2 依次固定为 `调查信息` 和 `调查报告`：

```markdown
# <调查主题>

## 调查信息
- 核心问题: <本文件持续追踪的问题链>
- 状态: 调查中 | 暂停 | 已结束
- 最新报告时间: 2026-07-21T14:05:30+08:00

## 调查报告

### <本轮报告的语义标题>
- 形成时间: 2026-07-21T14:05:30+08:00

#### 形成时背景
<本轮调查形成时的上下文、触发认识、已知事实、假设、未知、约束和必要基线>

#### 调查目的
<本轮要回答的问题、支持的判断和预定边界>

#### 调查范围与依据
<实际检查的对象、来源、时点或版本、方法、实际覆盖范围，以及未检查或不能覆盖的内容>

#### 调查结果与边界
<本轮形成的事实、推断、建议、动作状态、未知、适用条件和复核条件>
```

四个固定 H4 之后允许增加 `时间线`、`证据`、`对照`、`状态模型`、`恢复` 或其他有语义名称的 H4；是否增加由 [SKILL.md](../SKILL.md) 的场景判断决定。

### 调查信息

1. 三个字段按示例顺序各出现一次，每个值是非空单行列表项，可以包含普通 Markdown 行内语法。
2. `核心问题` 定义文件身份，不能只是宽泛主题名称。
3. `状态` 只表示是否继续调查：`调查中` 表示仍在推进或值得继续，`暂停` 表示暂不推进，`已结束` 表示当前不再计划继续；它不承诺问题已解决。
4. `最新报告时间` 逐字等于最后一份报告的 `形成时间`。状态或纯编辑变化不改变它。
5. 顶部不保存背景、目的、范围、依据或当前结果摘要，避免形成第二份事实来源。

### 完整报告

1. `调查报告` 只出现一次且至少包含一个 H3。标题概括本轮认识、对象或复查目的，不使用“第一次”“继续调查”等无语义阶段名。
2. 每份报告首个非空内容固定为 `- 形成时间: <timestamp>`。
3. 每份报告的前四个 H4 依次固定为 `形成时背景`、`调查目的`、`调查范围与依据`、`调查结果与边界`；各出现一次并包含实际内容，检查器按这些精确标题校验。
4. 四项核心的内容语义、独立阅读要求和人工审阅标准由 [SKILL.md](../SKILL.md) 承接。
5. 可选 H4 只能放在四个固定核心之后，不能替代固定核心。
6. `调查报告` 之后可以增加附录、术语等 H2；它们不替代完整报告。

## 时间与维护

1. `形成时间` 和 `最新报告时间` 使用带显式时区、无小数秒的 RFC 3339 时间戳。报告按形成时间非递减顺序追加。
2. 追加报告时写在 `调查报告` 容器末尾，并同步主题信息和索引。原地修正或只更新状态时不新增报告，也不改变最新报告时间。
3. 主题文件不进入物理归档目录；状态、报告序列和 Git 历史共同表达演进。

## 检查器

从本 skill 目录运行，或使用检查器的实际安装路径：

```text
node <investigation-report-skill>/scripts/check-investigations.mjs --root <workspace-root>
```

可选参数：

```text
--investigations-dir <path>  使用工作区内其他调查根目录
--topic <topic-id>           只检查一个主题，可重复
--report <relative-path>     只检查一个主题文件，可重复
```

`--report` 的值始终是相对调查根目录的主题文件路径。重复参数取并集；同时使用 `--topic` 和 `--report` 时取交集。反斜杠会归一化为 POSIX 分隔符。筛选没有命中时失败，调查根目录或索引无法读取时始终失败。

检查器机械验证：

1. 调查根目录、索引、主题目录、路径层级和 kebab-case 文件名。
2. 索引 H1、主题、条目、三个字段、重复项和目标文件。
3. 主题文件 H1、前两个固定 H2、调查信息、状态和至少一份 H3 报告。
4. 每份报告的形成时间；四个固定 H4 的存在、唯一、非空和顺序；报告时间顺序，以及最新报告时间一致性。
5. 主题文件与索引之间的标题、核心问题、状态、时间和路径投影。

检查器不判断章节语义、证据质量、场景义务、状态选择或拆分判断；这些由 `SKILL.md` 的形成与审阅流程承接。

退出状态：`0` 表示通过，`1` 表示结构或投影失败，`2` 表示参数错误。
