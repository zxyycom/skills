# 调查报告固定契约

本文件是 `investigation-report` 的唯一固定存储契约，承接调查根目录、索引、报告外壳、调查记录、状态、时间、更新与拆分规则，以及检查器的精确语义。这些固定项服务于快速汇报、接续查找和机械一致性；调查方法、证据质量、正文内容和人工审阅由 [SKILL.md](../SKILL.md) 承接。

## 内容 owner

1. 每份报告的 `调查概述` 投影当前起因、问题、范围、认识、状态和报告时间；`调查记录` 中的每个调查段承接一轮调查形成时的背景、方法、证据、结果、推断和未知。
2. `<investigation-root>/investigation-index.md` 只投影报告的标题、核心问题、状态和最近更新时间，用于发现报告，不承接完整认识或当前事实。
3. `scripts/check-investigations.mjs` 只读检查报告路径、固定 Markdown 结构、字段、形成时间和索引对应关系，不写入或重排文件。
4. 代码、配置、规范、指南、决策和测试材料继续承接各自当前事实与行为；报告通过链接引用这些 owner，不复制其长期契约。

## 调查根目录与稳定身份

默认调查根目录是工作区内的 `docs/investigations`：

```text
<investigation-root>/
├── investigation-index.md
└── <topic-id>/
    └── <semantic-slug>.md
```

1. `topic-id` 和 `semantic-slug` 使用英文小写、数字与连字符组成的 kebab-case 名词或名词短语。
2. 一个相对路径只承接一个核心问题链。文件名表达调查主题，不使用日期前缀、阶段、版本或工单号作为身份。
3. 报告创建后，相对调查根目录的路径保持稳定；标题澄清、状态变化、证据增加和重新开启不改变路径。
4. 同时涉及多个主题时选择主要主题。默认不为单份报告再建立目录；只有独立材料确实需要共同分发且普通链接无法清楚承接时，才由目标项目另行约定。
5. 调查根目录可以通过检查器的 `--investigations-dir <workspace-relative-path>` 改为工作区内其他位置；同一集合始终使用同一根目录。
6. 调查根目录在首次创建报告时建立并同时创建索引；除索引外的 Markdown 文件都必须是主题目录中的调查报告。

## 索引结构

索引首个非空行固定为：

```text
# 调查索引
```

每个主题使用二级标题，每份报告使用一个顶层列表项和三个嵌套字段：

```markdown
# 调查索引

## codex

- [项目 Shell MCP 注册调查](codex/project-shell-mcp-registration.md)
  - 核心问题: 为什么项目 Shell MCP 没有注册到可用工具列表？
  - 状态: 调查中
  - 最近更新时间: 2026-07-21T14:05:30+08:00
```

索引规则：

1. 主题标题逐字等于报告路径第一段；索引链接使用相对调查根目录的 POSIX 路径。
2. 链接文字逐字投影报告 H1 的纯文本；三个字段分别逐字投影报告概述中的 `核心问题`、`状态` 和 `最近更新时间`。
3. 每份报告有且只有一个索引条目，每个索引条目存在对应报告。
4. 一个条目只包含链接和三个非空字段；字段名称、顺序和合法值按示例固定。
5. 索引可以在 H1 与首个主题之间保留简短用途说明；主题内部不插入会打断条目结构的说明或额外层级。

## 报告结构

报告首个非空行是唯一一级标题，前两个二级标题依次固定为 `调查概述` 和 `调查记录`：

```markdown
# <调查主题>

## 调查概述
- 起因: <发现了什么问题，或为什么想了解>
- 核心问题: <本报告持续追踪的一个问题链>
- 调查范围: <包含的对象、时间或环境，以及必要排除边界>
- 当前认识: <目前知道什么，以及影响理解的主要未知>
- 状态: 调查中 | 暂停 | 已结束
- 首次形成时间: 2026-07-20T09:00:00+08:00
- 最近更新时间: 2026-07-21T14:05:30+08:00

## 调查记录

### <本轮调查主题>
- 形成时间: 2026-07-21T14:05:30+08:00

<按本轮问题展开背景、范围与方法、证据与结果、推断、未知和接续方向>
```

固定概述规则：

1. 七个字段按示例顺序各出现一次，每个值使用一个非空单行列表项；值内可以使用普通 Markdown 行内语法。
2. `起因` 写观察到的问题、异常、知识缺口或了解动机，不提前假定根因。
3. `核心问题` 定义报告身份。它可以包含共同服务于同一认识链的子问题，但不能只是宽泛主题名称。
4. `调查范围` 说明证据适用的对象和边界；版本、环境、样本或时间显著影响结果时在这里概括，并在调查段展开。
5. `当前认识` 是全部调查段当前支持内容的压缩投影，可以表达部分答案、条件性解释或尚无法判断，不要求形成结论。
6. `状态` 只表达是否继续调查，不表达问题是否解决或是否已经形成决定。
7. `首次形成时间` 是报告第一次形成可用于汇报和接续的调查段的时刻，创建空外壳、首次发现问题或更早的零散搜索不作为该时间。
8. `最近更新时间` 在调查范围、有效证据、当前认识或状态实质变化时更新；纯排版、措辞和链接修复不改变它。

调查记录规则：

1. `调查记录` 只出现一次且至少包含一个三级标题调查段；三级标题概括本轮形成的认识或调查对象，不使用单纯的“第一次”“继续调查”等阶段名。
2. 每个调查段的首个非空内容固定为 `- 形成时间: <timestamp>`。形成时间表示本轮背景、证据、结果和推断被整理成这段可接续内容的时刻，不代替实验、日志或外部来源自身的观测时间。
3. 每个调查段语义上覆盖本轮背景与局部问题、范围与方法、关键证据和结果、结果支持的推断、冲突或替代解释、未知，以及仍值得继续时的接续方向；标题、段落和更深层章节由问题决定。
4. 调查段是阶段性认识，不是原始命令日志。多条动作共同回答一个局部问题时合并为一段；新的背景、证据或认识形成可独立汇报的下一轮时再增加调查段。
5. 新调查段按形成顺序追加。较早调查段保留其形成时的背景和认识；后续证据修订它时，在新调查段说明变化，并同步顶部当前认识。纯文字或定位修复可以原地更正。
6. `调查记录` 之后可以按需增加附录、术语或其他二级章节；这些补充不替代调查段承担的语义。

时间规则：

1. `首次形成时间`、`最近更新时间` 和每个调查段的 `形成时间` 都使用带显式时区且不含小数秒的 RFC 3339 时间戳，例如 `2026-07-21T14:05:30+08:00` 或 `2026-07-21T06:05:30Z`。
2. `最近更新时间` 对应的实际时刻不得早于 `首次形成时间`；每个调查段的形成时刻不得早于首次形成时间或晚于最近更新时间。
3. 易变外部事实、实验、日志和运行观测仍在对应证据附近保留各自时间与条件，不能只用报告或调查段形成时间代替。

## 状态与演进

1. `调查中` 表示当前仍在推进或仍值得继续。
2. `暂停` 表示问题仍有调查价值，但当前缺少条件或暂不推进。
3. `已结束` 表示当前不再计划继续调查；它不表示问题已经解决、获得完整答案或形成长期决定。
4. 同一核心问题增加调查段、改变认识、暂停、结束或重新开启时原地更新报告和索引投影。
5. 新发现的问题能够脱离原问题独立调查、交接和结束时，在同一主题目录创建兄弟报告，并在相关正文中使用普通相对链接说明来源或联系。索引不维护正式关系图。
6. 调查报告不进入物理归档目录；状态、形成时间、正文和 Git 历史足以表达当前调查与版本演进。

## 检查器

从 skill 目录运行，或使用脚本绝对路径：

```text
node scripts/check-investigations.mjs --root <workspace-root>
```

默认检查所选调查根目录中的完整索引和全部报告。可选参数：

```text
--investigations-dir <path>  使用工作区内其他调查根目录
--topic <topic-id>           只检查一个主题，可重复
--report <relative-path>     只检查一份报告，可重复
```

筛选规则：

1. 同一参数重复时取并集；同时使用 `--topic` 和 `--report` 时取交集。
2. `--report` 相对调查根目录解析，命令行中的反斜杠会归一化为 POSIX 分隔符。
3. 筛选检查只校验选中报告及对应索引条目；无关报告和其他主题的结构问题不使本次筛选失败。
4. 调查根目录或索引无法读取仍然阻断筛选检查；筛选没有命中任何文件或索引目标时失败，避免拼写错误静默通过。

检查器机械验证：

1. 调查根目录、索引文件、主题目录、路径层级和 kebab-case 语义文件名；除索引外的每个 Markdown 文件都按报告检查，文件名是否实际表达日期、阶段或工单语义由人工判断。
2. 索引 H1、主题、条目结构、字段、重复项和目标文件。
3. 报告唯一 H1、前两个固定 H2、七个概述字段、状态、至少一个调查段、各调查段形成时间及报告时间先后关系。
4. 报告与索引之间的标题、核心问题、状态、最近更新时间和路径一一对应。

检查器不判断核心问题是否合理、调查段内容是否充分、来源是否可靠、推断是否成立、状态是否恰当、语义文件名是否合适或是否应该拆分；这些语义由 `SKILL.md` 的形成与审阅流程承接。

退出状态：`0` 表示所选范围结构通过，`1` 表示结构或对应关系失败，`2` 表示命令参数错误。
