# Investigation Report

## 目标

围绕一个能够持续追踪的核心问题稳定形成调查报告。新读者应能先从概述快速汇报起因、范围和当前认识，再从各轮调查恢复当时背景、方法、证据、结果、推断和未知，并据此继续探索而不必重新建立全部上下文。报告可以停在部分认识、条件性解释或明确未知，不以形成决策、修复方案或完整结论为完成前提。

## 使用条件

1. 用户明确要求记录、沉淀、创建、更新或审阅调查报告时使用本 skill，并在当前授权内形成或维护报告。
2. 常见适用内容包括：为实现某项能力调查候选库，研究同类或竞品，深入分析机制与性能，恢复实际执行逻辑，以及研究并发、时序或状态协作。这些只是常见报告对象；普通调查、排障或问答没有明确沉淀要求时，不主动建议、创建或修改调查报告。

## 内容 owner

1. 本文件承接报告导向的调查流程、每轮调查如何形成可接续内容、证据与推断质量、审阅和任务出口。
2. [investigation-report-contract.md](references/investigation-report-contract.md) 承接目录、索引、固定概述、调查记录外壳、状态、时间、更新与拆分规则以及检查器精确语义。
3. `scripts/check-investigations.mjs` 只读检查固定结构和索引投影，不判断问题是否清楚、证据是否可靠、分析是否成立或报告是否便于接续。
4. 代码、配置、规范、指南、测试和决策记录继续承接当前行为与长期选择；调查报告承接形成认识所需的背景、方法、证据、分析和边界。

## 主动读取

1. 先读取目标工作区指令、相关事实 owner、已有调查索引和可能相关的调查报告。
2. 创建、更新、拆分或结构审阅报告前，完整读取固定契约。
3. 只读审阅不修改报告；创建或更新报告只使用当前任务已经授予的调查与写入权限。
4. 调查关联 OpenSpec change、长期决策、测试证据或其他专职流程时，按当前任务读取相应 owner；这些能力不是本 skill 的安装前提。

## 主流程

### 1. 建立报告框架

1. 从发现的问题、异常、比较需求或知识缺口写出起因，不把尚未证明的原因写进问题定义。
2. 将报告收敛到一个能够持续追踪的核心问题。共同服务于同一认识链的子问题可以保留；读者、证据或结束条件能够独立的问题分别成文。
3. 把未来读者视为没有参与当前对话的人：明确他需要快速汇报什么、继续调查什么，并补足理解对象所需的最小背景、术语和关系，不依赖作者口头说明，也不复制已有事实 owner 的完整内容。
4. 说明证据适用的版本、环境、样本、时间范围和必要排除项。根据报告可能支持的后续行为确定证据门槛；安全、权限、兼容性或重大投入相关主张需要更直接、可复核的依据。
5. 搜索已有索引和报告。核心问题相同或只是增加一轮调查时更新原报告；标题不同但认识链相同不新建文件。

### 2. 确定本轮调查

1. 一轮调查是能够形成一段阶段性认识的工作单元，不按单条命令、单个来源或每次试错拆分。
2. 写清本轮为什么发生、要回答核心问题中的哪一部分、当前已知什么，以及哪些事实、候选解释或关键未知需要证据区分。
3. 为实验和测量确定对象、步骤、环境、关键参数、数据口径和比较基线；对易变事实确定观测时间、版本和范围。
4. 调查动作默认保持只读。实验、配置变化、进程操作或其他副作用只有在当前任务已经授权时执行，并保留动作与影响边界。

### 3. 取得并判断证据

1. 优先使用项目事实 owner、实际运行结果、可复现样本和一手外部来源；二手材料用于提供线索时说明局限。
2. 为重要事实保留可定位来源，例如文件与代码位置、命令及关键参数、日志时间窗、实验条件、数据口径或外部链接与访问时间。
3. 对重要材料说明它直接证明什么、不能证明什么，以及它与核心问题的关系；来源数量不能代替证明能力。
4. 区分已确认事实、基于证据的推断、有限范围内未观察到的现象和当前未知。冲突来源按时间、环境、版本、样本与口径比较，无法消解时保留冲突及其影响。
5. 区分相关性、时间先后、候选机制和因果证明；关键因果环节尚未覆盖时保留为推断或工作模型。
6. 提炼重复日志和原始输出，只保留复核主张所需的片段与定位方式；密钥、令牌、私人内容和无助于判断的本机标识不进入报告。

### 4. 形成本轮调查段

1. 在固定 `调查记录` 中新增一个三级标题调查段，并记录这段认识的 `形成时间`。该时间表示背景、证据、结果和推断被整理成当前段落的时刻，不代替证据自身的观测或访问时间。
2. 形成时间之后先写本轮能够直接复述的结果及其适用边界；暂时不能回答时，先写目前不能判断什么以及缺少哪类证据，不让读者读完过程后才发现本轮没有结论。
3. 随后围绕本轮局部问题组织必要背景、范围与方法、关键证据、结果支持的推断、冲突或替代解释、仍然未知的部分，以及值得继续时最有区分力的调查方向。按“主张—证据—解释—边界”把相关内容放在一起，不按搜索、试错和命令执行顺序堆积过程日志。
4. 根据对象选择表达方式：
   - 能力与库调研：目标和约束、比较口径、各候选证据、关键差异与适用边界；采用哪一项由决策 owner 承接。
   - 竞品调研：观察范围与时间、事实来源、关键机制或体验差异、可参考部分及其适用边界。
   - 性能分析：负载与基线、测量方法、结果、候选机制、反例以及当前能够支持到什么因果强度。
   - 机制、实际执行逻辑与并发研究：入口、调用或状态链、关键分支、数据与控制关系、时序和同步边界；关系复杂时使用表格、时间线或 Mermaid。
5. 一轮材料不足以形成强结论时，如实写出部分结果和未知。当前任务允许继续调查时补最有区分力的证据；否则让缺口和接续入口保持明确。

### 5. 刷新当前概述

1. 完成本轮调查段后再更新顶部 `调查概述`。`当前认识` 先给出核心问题目前最直接的答案，再压缩正文真正支持的主要条件和影响理解的未知。
2. `首次形成时间` 在报告第一次形成可用调查段时确定并保持不变；本轮带来有效证据、认识、范围或状态变化时更新 `最近更新时间`。
3. 概述应足以支持一次快速汇报：为什么调查、调查到哪里、当前知道什么、结论适用到哪里、是否还会继续。
4. 新一轮调查形成新的调查段，并在概述中投影最新认识。新证据改变既有认识时，在新段明确说明原认识被加强、削弱、推翻还是限缩；旧调查段保留当时背景与认识，不把报告维护成无法恢复演进原因的覆盖式摘要。纯文字、排版和链接修复直接修正原位置，不新增调查段或更新时间。
5. 新问题能够脱离原问题独立调查、交接和结束时创建兄弟报告；只是在同一认识链上补证时继续更新原报告。

### 6. 审阅报告质量

依次检查：

1. **快速汇报**：只读概述能否先给出当前最直接的答案，再准确说明起因、问题、范围、限制、主要未知和状态。
2. **降低上手门槛**：不了解本次过程的读者能否在不读取原对话、不询问作者且不重跑全部搜索与实验的情况下，恢复必要背景、对象关系、方法和关键证据。
3. **支持接续**：各轮形成时间、当时背景、已完成工作、认识相对前一轮的变化、未知和下一项有效调查入口是否清楚。
4. **问题对齐**：主要内容是否服务于核心问题，标题和概述是否把范围说得比证据更大。
5. **可追溯与校准**：关键事实能否定位来源、方法、时间和条件；事实、推断、未知与未观察到是否分开，措辞强度是否超过证据。
6. **解释与密度**：每轮是否先给可复述结果或明确无法判断，再说明证据怎样改变认识；冲突、反例和替代解释是否得到处理，关键认识是否被过程日志、重复背景或无关细节淹没。
7. **责任与隐私**：规范、决定、配置和行动是否留在各自 owner，敏感信息和不必要的私人路径是否排除。

审阅已有报告时，先说明它当前能够支持哪些汇报和后续探索、不能支持什么判断、接续调查卡在哪里；再指出会改变当前认识或使主张无法复核的必需补充、造成范围过宽或主张过强的收敛项，以及不影响可信度的可选增强。只读审阅不直接重写报告。

### 7. 整理与校验

1. 按固定契约选择主题语义路径，维护固定概述、调查记录和索引投影。首次创建报告且调查根目录不存在时，同时建立索引、主题目录和报告。
2. 新建报告、调整索引或批量维护后，默认运行完整检查：

   ```text
   node scripts/check-investigations.mjs --root <workspace-root>
   ```

3. 只修改单个主题或报告时，可以使用 `--topic <topic-id>` 或 `--report <relative-report-path>` 收窄检查；两者都可重复，精确语义见固定契约。
4. 检查器通过只证明固定结构、形成时间格式和索引投影一致；报告是否便于汇报、上手和接续仍按前述标准人工判断。

## 交接边界

1. 调查中发现需要长期采用的方向时，将已确认取舍交给决策 owner；调查报告可以作为背景来源，但不是决策附件或替代正文。
2. 需要形成实现计划时交给项目规划或 OpenSpec 流程；需要修改代码、配置或运行状态时按当前任务授权进入相应实施流程。
3. 发现稳定验证义务时交给测试策略或测试证据 owner。调查中的一次性命令和样本不因出现在报告里自动成为长期测试。
4. 下游是否存在不影响调查报告成立；没有决策、修复或后续任务也可以结束调查。

## 完成标准

### 创建或更新报告

1. 概述先给核心问题当前最直接的答案并能够独立支持快速汇报，调查边界和当前认识没有超出正文证据。
2. 每轮调查都有形成时间，先给可复述结果或明确无法判断，再恢复本轮背景、方法、关键证据、推断、未知和有价值的接续入口；修订既有认识时已经说明变化。
3. 新读者不依赖当前对话或作者补充就能理解对象和必要上下文，关键事实能够追溯到来源、方法、观测时间和条件，不需要重跑全部过程才能继续。
4. 事实、推断、替代解释和未知清楚分离，当前认识的强度与证据相称；正文没有退化为过程日志或原始材料堆积。
5. 报告没有扩张为未授权的决定或实施，没有泄露敏感信息，也没有复制其他 owner 的长期契约。
6. 固定概述、调查记录和索引满足契约；已运行默认完整检查或与改动范围相称的筛选检查，并报告实际结果。

### 审阅报告

1. 已说明报告当前能够和不能支持的汇报、理解与接续用途。
2. 已指出影响可信度、上手门槛或继续调查的具体缺口，没有越过只读授权。
