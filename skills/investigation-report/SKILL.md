# Investigation Report

## 目标

围绕一个能够持续追踪的核心问题维护稳定的主题文件，并在每次形成新认识时追加一份能够独立阅读、独立汇报和独立复核的完整报告。每份报告都明确当时理解结果所需的背景、为什么在此时调查，以及实际调查结果；读者不需要拼接旧报告或恢复原对话才能理解当前报告。最新一份报告表达当前认识，较早报告保留当时的完整快照和演进依据。

报告可以停在部分结果、条件性解释或明确未知，不以形成决定、修复方案或强结论为完成前提。

## 使用条件

1. 用户明确要求记录、沉淀、创建、更新或审阅调查报告时使用本 skill，并在当前授权内形成或维护报告。
2. 常见内容包括候选库调研、同类或竞品研究、机制与性能分析、实际执行逻辑恢复，以及并发、时序或状态协作研究。普通调查、排障或问答没有明确沉淀要求时，不主动建议、创建或修改调查报告。

## 内容 owner

1. 本文件承接报告导向的调查流程、背景与结果质量、证据和推断校准，以及何时追加、拆分或原地修正的行为判断。
2. [investigation-report-contract.md](references/investigation-report-contract.md) 承接目录、索引、主题文件结构、字段与章节顺序、状态与时间更新、报告排序以及检查器精确语义。
3. `scripts/check-investigations.mjs` 只读检查固定结构、三个核心章节、形成时间和索引投影，不判断背景是否充分、证据是否可靠或调查结果是否成立。
4. 代码、配置、规范、指南、测试和决策记录继续承接当前行为与长期选择；调查报告承接某一时点形成认识所需的上下文、证据、分析和边界。

## 主动读取

1. 先读取目标工作区指令、相关事实 owner、已有调查索引和可能相关的主题文件。
2. 创建、更新、拆分或结构审阅报告前，完整读取固定契约。
3. 只读审阅不修改报告；创建或更新只使用当前任务已经授予的调查与写入权限。
4. 调查关联 OpenSpec change、长期决策、测试证据或其他专职流程时，按当前任务读取相应 owner；这些能力不是本 skill 的安装前提。

## 主流程

### 1. 确定主题文件和本次报告

1. 将主题文件收敛到一个能够持续追踪的核心问题。共同服务于同一认识链的子问题可以保留；能够独立调查、交接和结束的问题分别成文。
2. 搜索索引和已有主题文件。核心问题相同、只是补充证据、复查现状或修订认识时，向原文件追加完整报告；标题变化不构成新文件理由。
3. 明确本次报告为何发生、希望查清什么以及可能支持什么后续理解。一次报告是能够形成阶段性认识的工作单元，不按单条命令、单个来源或每次试错拆分。
4. 把未来读者视为没有参与当前对话的人。最新报告不能依赖“同上”“沿用前文”或仅写相对上一版的变化；引用旧报告作比较时，也要重述理解当前结果所需的现状。

### 2. 恢复并写清背景

1. 背景写形成和解释本次结果所必需的当前上下文，例如项目处境、所处阶段、对象责任、现有机制、术语、已知状态和关键约束。它不是起因的别名，也不是来源清单。
2. 背景以本次报告形成时的状态为准。项目已经换阶段、实现或外部条件时，写新的当前背景，不让读者从旧报告拼出状态。
3. 只链接其他事实 owner 不能替代背景；报告先概括影响理解的事实，再给出可复核定位，避免复制 owner 的完整长期内容。
4. 判断版本、环境、样本、时间范围或工作区状态是否显著影响结果；影响时按固定契约记录可复核的条件性基线，不影响时不增加基线字段。已提交 hash 不能单独代表未提交状态，非 Git 对象也不强行使用 hash。

### 3. 取得并判断证据

1. 优先使用项目事实 owner、实际运行结果、可复现样本和一手外部来源；二手材料用于线索时说明局限。
2. 为重要事实保留可定位来源，例如文件与代码位置、命令及关键参数、日志时间窗、实验条件、数据口径，或外部链接与访问时间。
3. 对重要材料说明它直接证明什么、不能证明什么，以及它与核心问题的关系；来源数量不能代替证明能力。
4. 区分已确认事实、基于证据的推断、有限范围内未观察到的现象和当前未知。冲突来源按时间、环境、版本、样本与口径比较，无法消解时保留冲突及影响。
5. 区分相关性、时间先后、候选机制和因果证明；关键因果环节尚未覆盖时保留为推断或工作模型。
6. 调查动作默认保持只读。实验、配置变化、进程操作或其他副作用只有在当前任务已经授权时执行，并记录动作与影响边界。
7. 提炼重复日志和原始输出，只保留复核主张所需的片段与定位；密钥、令牌、私人内容和无助于判断的本机标识不进入报告。

### 4. 形成一份完整报告

1. 按固定契约在 `调查报告` 容器末尾建立新报告外壳；以下步骤负责写实三个核心章节的语义，不在入口重复字段与顺序规则。
2. `背景` 完整表达理解本次结果所需的当时上下文和条件；即使前一份报告背景相同，也不能省略或只写差异。
3. `起因` 写为什么在此时形成这份报告，不提前假定根因。它可以很简单，例如“距上次调查已久，需要了解当前状态”，但必须让读者知道本次调查为何发生。
4. `调查结果` 先直接回答本次实际查到了什么、适用于什么边界；暂时不能判断时，直接写明不能判断什么以及缺少哪类证据。结果可以包含多个相互关联的发现，不强迫形成单一结论。
5. 范围与方法、基线、证据、推断、结论、冲突或替代解释、未知和下一步只在有助于支撑或解释结果时增加，不替代三个核心章节；具体位置遵循固定契约。
6. 推论和结论必须能够回指报告中的调查结果与证据，并显式标明推断强度；不能把由背景假设出来的内容倒写成调查结果。
7. 按“主张—证据—解释—边界”组织相关内容，不按搜索、试错和命令执行顺序堆积过程日志。关系复杂时使用表格、时间线或 Mermaid。

### 5. 演进主题文件

1. 同一核心问题增加证据、复查现状、改变背景或修订认识时，在原主题文件末尾追加一份新的完整报告。
2. 新问题能够脱离原问题独立调查、交接和结束时，创建兄弟主题文件并按需说明联系。
3. 纯错别字、格式、失效链接或不改变语义的定位修复可以原地更正。只改变是否继续调查时可以只更新状态；状态变化来自重新调查或新认识时仍追加完整报告。
4. 追加时重述当前背景与调查结果，让最新报告承担当前口径并让旧报告保留形成时快照；完成正文后按固定契约同步调查信息和索引。

### 6. 审阅报告质量

依次检查：

1. **独立完整**：任取一份报告，读者能否只靠该报告恢复背景、调查动机和调查结果，而不必拼接前文或原对话。
2. **当前可用**：最新报告能否直接支持当前汇报，项目阶段、对象状态和适用边界是否仍明确。
3. **结果优先**：调查结果是否直接回答本次查到了什么或为什么暂时无法判断，没有被过程、推断或建议替代。
4. **可追溯与校准**：关键事实能否定位来源、方法、观测时间和条件；事实、推断、结论、未知与未观察到是否分开，措辞强度是否超过证据。
5. **演进可回放**：旧报告是否保留当时完整状态，最新报告是否重述当前背景和结果，实质变化是否通过追加而非覆盖表达。
6. **问题与 owner 对齐**：内容是否服务于核心问题；规范、决定、配置和行动是否留在各自 owner；敏感信息是否排除。

审阅已有报告时，先说明每份报告当前能够支持什么、最新报告能否作为当前口径、哪些结论因背景不足或证据缺失而不可靠；再区分必须补足的核心内容和可选增强。只读审阅不直接重写报告。

### 7. 整理与校验

1. 按固定契约选择主题语义路径，维护调查信息、完整报告和索引投影。首次创建且调查根目录不存在时，同时建立索引、主题目录和主题文件。
2. 新建主题文件、追加报告、调整索引或批量维护后，默认运行完整检查：

   ```text
   node scripts/check-investigations.mjs --root <workspace-root>
   ```

3. 只修改单个主题或文件时，可以使用 `--topic <topic-id>` 或 `--report <relative-report-path>` 收窄检查；两者都可重复，精确语义见固定契约。
4. 检查器通过只证明固定结构、三个核心章节、形成时间和索引投影一致；背景是否足以避免错误推断、证据是否可靠以及结果是否成立仍需人工判断。

## 交接边界

1. 调查中形成需要长期采用的方向时，将已确认取舍交给决策 owner；调查报告可以提供背景和证据，但不替代决策正文。
2. 需要形成实现计划时交给项目规划或 OpenSpec 流程；需要修改代码、配置或运行状态时按当前任务授权进入相应实施流程。
3. 发现稳定验证义务时交给测试策略或测试证据 owner。调查中的一次性命令和样本不因出现在报告中自动成为长期测试。
4. 下游是否存在不影响调查报告成立；没有决定、修复或后续任务也可以结束调查。

## 完成标准

### 创建或更新报告

1. 主题文件只承接一个稳定核心问题；同一问题的实质更新追加为完整报告，独立问题已拆为兄弟文件。
2. 每份报告都有形成时间和非空的背景、起因、调查结果，能够脱离旧报告与原对话独立阅读；最新报告足以表达当前认识。
3. 背景写清影响解释的当前项目处境、阶段和条件；对象基线只在影响结果时出现并足以复核实际调查状态。
4. 调查结果先给事实性发现或明确未知，推断和结论能够回指结果与证据，强度与证据相称。
5. 关键事实可追溯到来源、方法、时间和条件；正文没有退化为过程日志，也没有扩张为未授权的决定或实施。
6. 调查信息、报告顺序和索引满足固定契约；已运行与改动范围相称的检查并报告实际结果。

### 审阅报告

1. 已说明核心三项是否完整、最新报告能否作为当前口径，以及报告当前能够和不能支持的判断。
2. 已指出会导致错误结论、妨碍复核或阻断接续的具体缺口，没有越过只读授权。
